Monitoring and Detection

TAPs, collectors, and vulnerability scanners give security teams continuous visibility into network activity and a way to find weaknesses before attackers exploit them.

overview

Monitoring and detection involves continuously observing network activities to identify security incidents, anomalies, and vulnerabilities. The three tool categories are TAPs (traffic capture), collectors (aggregation and analysis), and vulnerability scanners (proactive weakness identification).

The chain is: TAPs capture raw traffic without disrupting it, collectors aggregate that data alongside logs from other sources, and vulnerability scanners independently probe systems for known weaknesses before an incident occurs.

TAPs (test access points)

A TAP copies data traveling between two network points and sends that copy to a monitoring device in real time, without interrupting normal traffic flow. A hardware TAP has three ports: two connecting network segments (e.g. modem and router) and one connecting to the monitoring device.

Passive TAP
Splits the signal and sends a copy to the monitoring device. Requires no power. Simple, reliable, and adds no latency. However, if the connection fails, a passive TAP cannot capture data. It also forwards malformed or damaged packets as-is to the monitoring device, since it performs no processing.
Active TAP
Physically interrupts the signal, regenerates it, then sends it to both the next network node and the monitoring device. Requires its own power supply. Can maintain data capture during network link issues if a backup power supply is present. Adds a small amount of latency. Without backup power, an active TAP becomes a single point of failure in the network.

TAP vs SPAN port vs cloud

  • Permanent installations (e.g. deploying a NIDS): TAPs are preferred because they provide full-fidelity capture of all traffic at all times
  • Temporary troubleshooting: a mirrored or SPAN port on a switch is more convenient than installing a physical TAP
  • Cloud environments: physical TAPs do not work in virtualized infrastructure; monitoring requires a virtual private cloud solution instead

collectors

Collectors gather network traffic and log data from multiple sources (servers, endpoints, firewalls) and forward it to monitoring tools for analysis. They form the backbone of an effective monitoring strategy by centralizing data from across the environment.

ToolTypeWhat it collects and does
Wireshark Protocol analyzer Captures and analyzes raw network packets in real time; useful for deep packet-level inspection and troubleshooting
Splunk Log aggregator / SIEM Collects log data from diverse systems and devices; provides real-time insights and anomaly detection across the environment
SolarWinds NPM Network performance monitor Monitors network traffic and performance metrics; alerts administrators to unusual activity such as bandwidth spikes

vulnerability scanners

Vulnerability scanners proactively probe systems and applications to identify known weaknesses before they can be exploited. They produce detailed reports that help administrators prioritize remediation by risk level.

ToolTypeKey characteristics
OpenVAS Open-source Greenbone's scanner; originally forked from Nessus before Nessus went commercial. Organized around NVT families. Supports custom scan configurations.
Nessus Commercial (Tenable) Extensive plugin library, easy to use, widely deployed. Covers outdated software, misconfigurations, and known CVEs with actionable recommendations.
Qualys Cloud-based Integrates with many security tools; offers vulnerability scanning, compliance checking, and continuous monitoring. Scales well for large, complex environments.

OpenVAS NVT families

OpenVAS organizes its tests into NVT families (Network Vulnerability Test families): groups of individual tests that focus on a specific category of risk, such as web servers, denial-of-service, or databases.

  • Each NVT family contains multiple individual tests targeting vulnerabilities within that category
  • Custom scan configurations let administrators target specific families (e.g. web servers only) to make scans more efficient and reduce noise
  • Individual NVTs within a family can be enabled or disabled, giving fine-grained control over what gets tested
  • Each NVT displays a CVSS score: a standardized risk rating that indicates how critical the vulnerability is, helping teams prioritize what to fix first
exam tips

key takeaways

  • TAPs copy traffic without interfering with it. Passive TAPs need no power and add no latency. Active TAPs regenerate the signal and can handle link failures, but can become a single point of failure without backup power.
  • TAPs for permanent monitoring; SPAN ports for temporary troubleshooting. Physical TAPs do not work in cloud environments.
  • Collectors centralize data from multiple sources. Splunk is the SIEM/log aggregator example; Wireshark is the packet-level capture tool.
  • OpenVAS was forked from Nessus when Nessus went commercial. OpenVAS is open-source; Nessus is commercial; Qualys is cloud-based.
  • CVSS score = risk level / criticality rating. Vulnerability scanners use it to help prioritize remediation.
  • NVT families in OpenVAS group tests by category (web servers, DoS, databases). Custom scan configs let you target specific families to reduce scan time.
quick decision rules
  • Captures traffic without disrupting the network → TAP
  • No power, no latency, but loses data on link failure → Passive TAP
  • Handles link failures, but adds latency and needs power → Active TAP
  • Temporary traffic capture without hardware → SPAN / mirrored port
  • Log aggregation, SIEM, anomaly detection → Splunk (collector)
  • Packet-level capture and analysis → Wireshark (collector)
  • Open-source vulnerability scanner, NVT families → OpenVAS
  • Cloud-based scanner, compliance checking, enterprise scale → Qualys

Resilient System Design

An overview of the security architecture techniques used to design systems that withstand disruption, detect threats efficiently, and recover quickly.

overview

Resilient system design covers Domain 2: Security Architecture. The core requirement is the ability to analyze requirements and design systems that remain operational under adverse conditions, including attacks, failures, and surges in demand.

This section covers six areas: security devices (the layered defense stack), monitoring and detection (observing the network), network traffic management (directing and optimizing flow), application layer security (protecting Layer 7), availability and scaling (handling load), and deployment and recovery strategies (geographic placement and restoration).

six topic areas

  1. Security Devices Firewalls, IDS, IPS, VPNs, and NAC form the layered defense that filters traffic, detects threats, encrypts communication, and controls access.
  2. Monitoring and Detection TAPs, collectors, and vulnerability scanners provide continuous visibility into network activity and identify weaknesses before they can be exploited.
  3. Network Traffic Management Forward proxies, reverse proxies, and CDNs control and optimize data flow across the network, applying policies and reducing latency at scale.
  4. Application Layer Security API gateways and WAFs protect applications at Layer 7 (OSI application layer) against API abuse, SQL injection, and cross-site scripting.
  5. Availability and Scaling Load balancing with persistent or non-persistent affinity keeps systems accessible. Vertical and horizontal scaling ensure capacity meets demand.
  6. Deployment and Recovery Strategies Geographical deployment distributes infrastructure to reduce latency and improve redundancy. Backups and failover mechanisms restore operations after disruption.

security devices

Security devices form a layered defense. Each device plays a distinct role, and they are designed to complement one another rather than serve as standalone solutions.

DeviceRoleKey detail
Firewall Filters traffic via Access Control Lists (ACLs) Establishes the network perimeter; controls what can enter and leave
IDS Detects and alerts on malicious activity Passive only — does not block traffic. Monitors and generates alerts.
IPS Detects, alerts, and actively blocks threats Active — extends IDS by mitigating detected threats in real time
VPN Encrypts data across untrusted networks Secures remote and site-to-site communication
NAC Enforces device health and security policies Checks compliance before granting network access; prevents non-compliant devices from connecting

monitoring and detection

Monitoring and detection provides continuous visibility into network activity so that threats can be identified and addressed before they cause damage.

TAPs (Test Access Points)
Hardware or software devices that capture all network traffic in real time. TAPs operate passively — they copy traffic without interfering with normal network operations, making them suitable for high-fidelity capture in production environments.
Collectors
Systems that aggregate and process data captured by TAPs and other sources. Collectors centralize data for further analysis, enabling identification of patterns, anomalies, and indicators of compromise across large volumes of traffic.
Vulnerability Scanners
Tools that proactively identify known technical weaknesses in systems and applications. Scanners allow organizations to address security gaps before they can be exploited, rather than discovering vulnerabilities only after an incident.

network traffic management

Network traffic management tools control, direct, and optimize data flow. Proxies mediate connections between clients and servers; CDNs solve the problem of delivering content efficiently at global scale.

ToolPositionDirection and purpose
Forward proxy Internal gateway (LAN side) Applies control policies to outbound internal requests before forwarding them to external destination servers
Reverse proxy Public interface (internet-facing) Receives inbound external requests, applies policies, and forwards them to the appropriate internal resources
CDN Globally distributed edge nodes Caches content at locations close to end users, reducing latency and improving load times worldwide. Built on distributed reverse proxies.

application layer security

Application layer security operates at Layer 7 of the OSI model, the application layer itself. It protects APIs and web applications using device-specific, granular rule sets not possible at lower layers.

API Gateway
Manages and secures API requests, handling authentication, rate limiting, and controlled data delivery to backend services. An API gateway acts as the front door to internal services exposed via APIs, enforcing access controls and preventing abuse before requests reach the backend.
WAF (Web Application Firewall)
Filters and monitors web application traffic to protect against common attacks such as SQL injection and cross-site scripting (XSS). Because a WAF is implemented at the application itself, it can apply highly specific rule sets tailored to the application's exact traffic patterns and known attack vectors.

availability and scaling

load balancing and affinity

Load balancing distributes incoming network traffic across multiple servers so that no single server becomes a bottleneck. Affinity controls how sessions are routed within that distribution.

  • Persistent affinity (session affinity): routes a user's session consistently to the same server, maintaining session state. Required when the application stores state locally on the server.
  • Non-persistent: any available server can handle each request, promoting better load distribution. Complicates state management but enables more flexible scaling.
  • Interoperability: diverse systems and applications communicate and function together seamlessly within the load-balanced environment, maintaining high availability.

scaling

TypeAlso calledMethodAdds
Vertical scaling Scale up Increase capacity of an existing machine More CPU, RAM, or storage to one server

deployment and recovery strategies

recovery

Recovery strategies ensure systems can be restored to operational status quickly after disruption.

  • Backups: regular copies of data that can be restored in the event of corruption or loss
  • Failover: standby systems that automatically take over if the primary system fails, minimizing downtime and maintaining continuity

geographical deployment

Placing infrastructure across multiple physical locations reduces latency, improves redundancy, and optimizes performance by serving users from nearby nodes.

  • Distributes load and eliminates single points of geographic failure
  • Introduces compliance complexity: infrastructure in each location is subject to the local laws and regulations of that jurisdiction, which dictate how data may be stored and processed
exam tips

key takeaways

  • IDS vs IPS: passive vs active. IDS detects and alerts only. IPS detects, alerts, and actively blocks. This distinction appears on nearly every exam that covers security devices.
  • Forward proxy = outbound; reverse proxy = inbound. Forward proxy sits inside the LAN and controls outgoing traffic. Reverse proxy sits on the public interface and controls incoming traffic.
  • CDNs are built on distributed reverse proxies. They solve the latency problem by caching content at edge nodes globally.
  • WAFs operate at Layer 7 and use application-specific rule sets. This allows them to block SQL injection and XSS in ways that a standard firewall cannot.
  • Vertical scaling = scale up (bigger machine); horizontal scaling = scale out (more machines). Know both terms and their synonyms.
  • Persistent affinity routes to the same server; non-persistent allows any server. Persistent is required for stateful applications; non-persistent suits stateless workloads.
  • Geographical deployment introduces jurisdictional compliance complexity. Each location's local laws govern how data in that facility can be stored and processed.
quick decision rules
  • Detects and alerts, no blocking → IDS
  • Detects, alerts, and blocks → IPS
  • Controls outbound internal traffic → Forward proxy
  • Controls inbound external traffic → Reverse proxy
  • Protects APIs (auth, rate limiting) → API gateway
  • Protects web apps (SQL injection, XSS, Layer 7) → WAF
  • Add capacity to existing machine → Vertical scaling (scale up)
  • Add more machines to pool → Horizontal scaling (scale out)

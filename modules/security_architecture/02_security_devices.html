Security Devices

A deep dive into the hardware and software tools that protect networks through traffic filtering, threat detection, encrypted tunneling, and access control enforcement.

overview

Security devices form a layered defense. Firewalls filter traffic using ACLs. IDS and IPS detect threats, with IPS adding active blocking. VPNs create encrypted tunnels for secure remote access. NAC enforces device health checks before granting access to the network.

Each device type has variants with meaningful differences. For the exam, knowing the specific capability of each variant (packet-filtering vs stateful vs NGFW, IDS vs IPS, persistent vs agentless NAC) is more important than memorizing brand names.

firewalls

Firewalls use Access Control Lists (ACLs) to permit or block traffic based on rules that define traffic type, source, destination, and the action to take. While routers can implement basic ACLs, dedicated hardware and software firewalls are far more effective at managing access control at scale.

TypeInspection approachKey limitation or capability
Packet-filtering Header only (IP address, port) Fastest and simplest, but cannot detect IP spoofing or session hijacking since it does not look beyond the header
Stateful Tracks active connection state Allows expected return traffic, blocks unsolicited traffic. More secure than packet-filtering.
Proxy (circuit-level) Session layer (L5) Acts as intermediary at the session layer; less inspection depth than application-level proxy
Proxy (application-level) Application layer (L7) Deepest inspection, most precise filtering; performance cost due to in-depth analysis
Proxy (kernel) Multiple OSI layers Filters across layers with minimal delay; balances security and performance. Less common.
Next-generation (NGFW) Application-aware, deep packet inspection Identifies and controls specific applications, not just ports; integrates IPS capabilities
WAF Web application traffic (HTTP) Targets SQL injection and XSS; can be inline (active blocking) or out-of-band (detection only)

IDS and IPS

Both IDS and IPS monitor traffic for suspicious activity. The critical difference is their response: an IDS only observes and alerts, while an IPS observes, alerts, and actively blocks.

IDS (Intrusion Detection System)
Passive. Detects, logs, and alerts on malicious activity. Does not take any action to stop it. An IDS is sometimes described as a burglar alarm: it tells you something is wrong but does not stop the intruder. Because it is passive, it does not add latency to traffic.
IPS (Intrusion Prevention System)
Active. Detects, logs, alerts, and blocks or mitigates the threat in real time. An IPS is placed inline with traffic, meaning all traffic passes through it. This allows it to drop malicious packets, but it also means a misconfigured IPS can block legitimate traffic.

detection methods

  • Signature-based: recognizes known attack patterns. Highly accurate for known threats, but cannot detect novel attacks that have no signature yet.
  • Anomaly-based: establishes a baseline of normal behavior and flags deviations. Can catch new, unknown threats but tends to produce more false positives.

VPN

A VPN creates an encrypted tunnel over an untrusted network such as the internet, allowing users and sites to communicate securely. The tunnel uses strong encryption (AES-256) between the user's device and the VPN concentrator at the receiving end.

Remote Access VPN (client-to-site)
Connects an individual remote user to the enterprise network. Ideal for remote workers and traveling employees who need access to internal resources like file servers. Can be strengthened with multi-factor authentication in addition to a username and password.
Site-to-site VPN
Connects two office locations to each other over the internet. A cost-effective alternative to dedicated leased lines between offices. Routers on both ends use encryption keys to keep all traffic secured in transit across the untrusted internet.

NAC

Network Access Control prevents unauthorized or non-compliant devices from joining the network. When a device attempts to connect, the NAC system performs a health check (antivirus status, patch level) before granting access. This applies to both on-premises and VPN-connected remote devices.

  • Device passes health check: granted full network access
  • Device fails health check: quarantined to a remediation area where it can receive updates but cannot reach the production network
  • VPNs and NAC are often combined: the VPN checks device compliance before granting full internal access

NAC agent types

Persistent agent
Software permanently installed on the device. Provides deep, reliable scanning. Best suited for corporate environments where the organization controls the hardware and can enforce agent installation policy.
Non-persistent agent
Temporarily installed when the device connects, deleted automatically once the health scan is complete. Common in environments with many guest or unmanaged devices, such as college campuses. Provides a scan without requiring permanent software on the device.
Agentless
No software installed on the individual device. Scanning is performed from a domain controller. Ideal for BYOD (Bring Your Own Device) environments where the organization cannot mandate software installation. Scans are less deep than agent-based solutions, but agentless capabilities continue to improve.
exam tips

key takeaways

  • IDS = passive (detect and alert only); IPS = active (detect, alert, and block). This is the most tested distinction in this topic area. Do not confuse them.
  • Packet-filtering firewalls inspect headers only and cannot stop IP spoofing or session hijacking. Stateful firewalls track connection state and are more secure.
  • NGFWs are application-aware. They go beyond port and protocol to identify specific applications and integrate IPS functionality with deep packet inspection.
  • WAF = web traffic specifically. It targets SQL injection and XSS. Can be inline (blocks) or out-of-band (detects only).
  • Signature-based detection = known threats only; anomaly-based = can catch new threats but more false positives. Both methods are used together in practice.
  • NAC quarantines non-compliant devices to a remediation area rather than simply blocking them. Persistent agents suit corporate hardware; agentless suits BYOD.
  • AES-256 is the encryption standard used in VPN tunnels. Know that client-to-site = remote access; site-to-site = office-to-office.
quick decision rules
  • Detects and alerts, no blocking → IDS
  • Detects, alerts, and blocks → IPS
  • Most basic firewall, header inspection only → Packet-filtering
  • Tracks connection state, allows expected return traffic → Stateful firewall
  • Application-aware, deep packet inspection, integrates IPS → NGFW
  • SQL injection, XSS, HTTP traffic → WAF
  • Remote worker connecting to enterprise network → Remote access VPN (client-to-site)
  • Connecting two office locations → Site-to-site VPN
  • BYOD, no software on device → Agentless NAC

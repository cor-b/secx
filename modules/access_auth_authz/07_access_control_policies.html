Access Control Policies

Rules that go beyond identity — evaluating context like location, device, and time before granting access, with separate systems to decide and enforce.

overview

Access control policies define the rules for granting or denying access to resources. Three components work together: conditional access enforces policies based on real-time conditions beyond just identity (location, device, time), policy decision points evaluate whether a request satisfies defined policies and make the allow/deny call, and policy enforcement points act on that decision by granting or blocking access. The PDP decides; the PEP enforces.

component 01

Conditional Access

Policies that evaluate contextual factors beyond identity: location, device type, network, time of day. Access may require MFA or be blocked entirely depending on conditions.

component 02

Policy Decision Point

The brain. Evaluates the access request against defined policies and makes the allow or deny decision. Checks role, location, device, and other factors against the policy ruleset.

component 03

Policy Enforcement Point

The gatekeeper. Acts on the PDP's decision by physically granting or blocking access to the resource. Does not decide — only enforces what the PDP determined.

conditional access

Traditional access control checks identity and role. Conditional access adds a layer of context: the same user may be granted full access from a managed corporate device on the internal network, required to complete MFA when connecting remotely, or blocked entirely when accessing from an unrecognized location on a personal device.

conditions evaluated

  • Location: is the user on the corporate network, a known VPN, or an unfamiliar region?
  • Device type: is the device a managed corporate device, a personal device, or a device with no MDM enrollment?
  • Time of access: is the request within working hours or from an unusual time that warrants additional scrutiny?
  • User risk: has the account shown suspicious behavior (password spray attempts, impossible travel) recently?

Microsoft Azure Active Directory Conditional Access

Azure AD Conditional Access is the leading enterprise tool for this. Administrators configure policies like: "require MFA when accessing financial systems from outside the corporate network" or "block access entirely if the device is not enrolled in Intune." The policy evaluates in real time at the moment of each access request.

policy decision points (PDP)

A policy decision point is the component that evaluates whether an access request satisfies the organization's security policies. When a user tries to access a resource, the PDP receives information about the request (who, from where, on what device, at what time) and evaluates it against the defined ruleset. The outcome is a binary decision: permit or deny.

  • The PDP does not enforce the decision — it only makes it
  • PDPs may consult multiple policy sources: identity claims, environmental data, resource classification
  • Example: user requests access to work email outside business hours. PDP checks the policy, determines this violates the time-of-access condition, returns a deny decision

AWS Identity and Access Management (IAM) functions as a policy decision point — it evaluates access requests to AWS resources against defined IAM policies and determines whether to allow or deny the action.

policy enforcement points (PEP)

A policy enforcement point is the component that acts on the PDP's decision. Once the PDP determines permit or deny, the PEP carries it out. The PEP sits between the user and the resource — it is the actual gate. If the PDP says deny, the PEP blocks access. If the PDP says permit, the PEP opens the path to the resource.

  • The PEP does not make decisions — it only enforces what the PDP determined
  • Multiple PEPs may exist in a single architecture (one at the network boundary, one at the application layer)
  • Example: PDP denies a request from an unrecognized location. The PEP prevents the user from connecting to the sensitive system, returning an access denied response

Palo Alto Networks Next-Generation Firewall is a classic policy enforcement point — it receives allow/deny decisions and enforces them at the network layer by permitting or blocking traffic flows.

EXAM TIPS

what to know for the exam

  • Conditional access = policies that consider context beyond identity (location, device, time). Stricter conditions trigger MFA or a full block.
  • Azure AD Conditional Access is the go-to enterprise tool for conditional access policies.
  • PDP = decision maker. Evaluates the request against policy. Returns permit or deny. Does not enforce.
  • PEP = enforcer. Acts on the PDP's decision. Grants or blocks actual access to the resource. Does not decide.
  • AWS IAM = PDP example. Palo Alto NGFW = PEP example.
  • The PDP and PEP work together: PDP says "deny," PEP blocks the connection. PDP says "permit," PEP opens the path.
decision rules
  • Scenario involves blocking access to sensitive data when a user is off the corporate network → conditional access
  • Scenario involves requiring extra authentication based on the device type being used → conditional access
  • Scenario involves a system evaluating whether an access request complies with defined security policies → policy decision point (PDP)
  • Scenario involves a system that blocks or allows a connection based on a prior policy evaluation → policy enforcement point (PEP)
  • Question asks which component makes the access decision → PDP
  • Question asks which component carries out the access decision → PEP
  • Question names AWS IAM and asks what role it plays → policy decision point
  • Question names a next-generation firewall and asks what role it plays → policy enforcement point

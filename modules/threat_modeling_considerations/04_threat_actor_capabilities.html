Threat Actor Capabilities

What attackers can actually do. Four escalating capabilities: knowing the target, planting weaknesses, building exploits, and compromising the supply chain to reach the target indirectly.

overview

Threat actor capabilities are the skills, tools, and expertise to find and exploit vulnerabilities. Four capabilities: knowledge (understanding the target via reconnaissance), vulnerability creation (introducing new weaknesses), exploit creation (building tools to leverage vulnerabilities), and supply chain access (compromising third-party vendors to reach the target indirectly).

capability 01

Knowledge

Understand the target via reconnaissance: passive OSINT then active scanning (Nmap). Maps weak points. Stealth needs evasion skills.

capability 02

Vulnerability Creation

Introduce new weaknesses (insert malicious code into open-source, social manipulation). Patient, long-term, high skill.

capability 03

Exploit Creation

Build tools to leverage vulnerabilities; advanced = zero-days. Needs programming, deep vuln knowledge, extensive testing.

capability 04

Supply Chain Access

Compromise a trusted vendor to reach the target indirectly (SolarWinds). Highly sophisticated, can hit many orgs at once.

knowledge

Knowledge is understanding the target's systems, defenses, and vulnerabilities via reconnaissance or insider sources. It begins with passive reconnaissance (OSINT, gathering public records, social media, website details without interacting), then shifts to active reconnaissance (directly scanning public interfaces, e.g. Nmap). Sophistication depends on the desired stealth: basic scans are quick, but stealthy methods require evasion skills (encrypting communications, spoofing IPs).

vulnerability creation

Vulnerability creation is the ability to introduce new weaknesses into a system, e.g. inserting malicious code into an open-source project the target uses, or physical/social manipulation (convincing an employee to change a config). It requires strong programming and system-configuration skills plus deep target knowledge, and it's a patient, long-term approach, planting flaws that go unnoticed but can be exploited later.

exploit creation

Exploit creation is developing tools or techniques to take advantage of known or newly discovered vulnerabilities (e.g. code to exploit a buffer overflow). Basic exploits use available scripts; advanced ones, like zero-day exploits for undisclosed vulnerabilities, require deep software knowledge, custom payloads, and months of research, usually reserved for well-resourced actors.

supply chain access

Supply chain access is an advanced capability targeting third-party vendors to indirectly compromise a primary target, inserting malware into legitimate software updates and bypassing the target's defenses. The canonical example is SolarWinds: attackers compromised the Orion platform and distributed a backdoor through trusted updates to numerous government and corporate clients, undetected for months. It requires deep knowledge of supply-chain relationships and vendor security, and can infiltrate many organizations at once. Mitigate by vetting vendor security, strict update verification, and monitoring third-party integrations.

EXAM TIPS

what to know for the exam

  • Four capabilities: knowledge, vulnerability creation, exploit creation, supply chain access (increasing sophistication).
  • Knowledge = recon: passive OSINT then active scanning (Nmap).
  • Vulnerability creation = planting new weaknesses; exploit creation = building tools (zero-days are advanced).
  • Supply chain access = compromising a trusted vendor to reach the target indirectly (SolarWinds).
  • Distinguish vulnerability creation (introduce the weakness) from exploit creation (build the tool to use it).
decision rules
  • Scenario gathers OSINT and scans with Nmap → knowledge
  • Scenario plants malicious code in an open-source library → vulnerability creation
  • Scenario writes a zero-day exploit → exploit creation
  • Scenario backdoors a vendor's update to reach clients → supply chain access

Attack Patterns

The tactics modeled in threat analysis: forcing malicious input into a system, breaking through access controls, and intercepting traffic in transit.

overview

Attack patterns are tactics and techniques threat actors use to exploit vulnerabilities, modeled in threat analysis. Three patterns: injection (inserting malicious code via input points), authentication and authorization (targeting access controls), and on-path (intercepting/altering communications, formerly man-in-the-middle).

pattern 01

Injection

Insert malicious code via input points (SQLi, command, code, XSS). Fix: input validation, parameterized queries, WAF.

pattern 02

Auth & Authz

Exploit access controls: weak password resets, session hijacking, broken RBAC. Fix: MFA (incl. resets), secure sessions.

pattern 03

On-Path

Intercept/alter traffic (formerly MITM): packet sniffing, session hijacking on weak Wi-Fi. Fix: TLS, certificates, monitoring.

injection

Injection attacks insert malicious code or commands via vulnerable input points. The most common is SQL injection (harmful SQL through input fields, e.g. ' OR 1=1;-- in a login form to bypass authentication), plus command injection, code injection, and cross-site scripting. They work when input isn't properly sanitized. Mitigate with rigorous input validation, parameterized queries (prepared statements) that separate code from data, and web application firewalls.

authentication and authorization

These attacks exploit weaknesses in access controls to bypass login or gain unauthorized permissions. Tactics include exploiting weak password reset mechanisms (using OSINT-discoverable info like email or knowledge-based questions), insecure access control lists, and broken authentication flows like session hijacking. Once in, an attacker may exploit flawed role-based access controls to escalate privileges. Mitigate with MFA, including for the password reset process, secure session management/strong session IDs, and properly configured RBAC.

on-path

On-path attacks (formerly man-in-the-middle) intercept communications between two parties, letting attackers eavesdrop on or alter data. Common types: packet sniffing (capturing traffic) and session hijacking (taking over an active session). A typical attack intercepts traffic on unprotected public Wi-Fi, capturing session tokens or credentials to hijack a banking session. Mitigate with encryption (TLS), certificates to verify party authenticity, and network monitoring/anomaly detection.

EXAM TIPS

what to know for the exam

  • Three modeled patterns: injection, authentication/authorization, on-path.
  • Injection (SQLi ' OR 1=1;--) → fix with input validation and parameterized queries.
  • Auth/authz attacks target weak password resets, session hijacking, broken RBAC → fix with MFA and secure sessions.
  • On-path = formerly man-in-the-middle; intercept/alter traffic (packet sniffing, session hijacking) → fix with TLS and certificates.
  • Public Wi-Fi without encryption is the classic on-path scenario.
decision rules
  • Scenario submits ' OR 1=1;-- in a login field → injection (SQLi)
  • Scenario abuses a weak password-reset flow → authentication/authorization attack
  • Scenario intercepts session tokens on public Wi-Fi → on-path attack
  • Scenario defends with parameterized queries → injection mitigation

Initial Access and Escalation Methods

The first half of the attack lifecycle: getting in (injection), grabbing credentials once inside (credential dumping), and climbing to higher privileges (escalation).

overview

Initial access and escalation methods are the TTPs attackers use to first get into a system and then elevate privileges. Three methods: injections (insert malicious code for initial access), credential dumping (extract stored usernames/passwords), and privilege escalation (exploit vulnerabilities/misconfigs to gain higher access).

method 01

Injections

Insert malicious code for initial access (SQLi via sqlmap, command injection). Fix: parameterized queries, input validation, WAF, least privilege.

method 02

Credential Dumping

Extract stored credentials/hashes/tickets from memory (Mimikatz). Fix: disable LM/NTLM, MFA, limit memory reads.

method 03

Privilege Escalation

Climb to higher access (Metasploit). Vertical (to admin/root) or horizontal (peer accounts). Fix: patch, least privilege, monitor.

injections

Injections insert malicious code to gain access or run unauthorized actions, SQL injection (interacting with the backend database in unintended ways) and command injection (running arbitrary code on the server). Attackers automate SQLi with tools like sqlmap to find flaws and retrieve/modify/delete data. Prevent with parameterized queries, input validation, WAFs, and least-privilege database configuration.

credential dumping

Credential dumping extracts stored usernames and passwords from compromised systems so attackers can impersonate legitimate users and bypass security. The well-known tool is Mimikatz, which extracts plaintext passwords, hashes, and Kerberos tickets from memory in Windows environments, enabling lateral movement and escalation. Prevent by disabling outdated protocols (LM/NTLM) in favor of stronger encryption, implementing MFA, and limiting the ability to read from memory (fewer admin rights, secure password policies).

privilege escalation

Privilege escalation exploits vulnerabilities to increase access rights, a critical step for unauthorized high-privilege actions. It's vertical (gaining admin/root) or horizontal (gaining peer accounts with similar access). Attackers use Metasploit to exploit escalation flaws (outdated software, misconfigured permissions). Prevent by patching, applying least privilege, and monitoring for unusual behavior (accessing sensitive files, elevated-privilege commands).

the demo: chained exploitation

The lesson demoed chaining these methods against a Metasploitable 2 target: an Nmap scan found a vulnerable VSFTPD 2.3.4 service, a Metasploit exploit opened a root shell (upgraded to a Meterpreter shell), then the /etc/passwd and /etc/shadow files were downloaded, combined with unshadow, and cracked with John the Ripper, demonstrating credential dumping and password cracking after initial access.

EXAM TIPS

what to know for the exam

  • Three methods: injections (initial access), credential dumping (steal credentials), privilege escalation (climb).
  • Mimikatz dumps plaintext passwords, hashes, and Kerberos tickets from memory (Windows).
  • Privilege escalation is vertical (to admin/root) or horizontal (to peer accounts).
  • Tools: sqlmap (SQLi), Mimikatz (credential dumping), Metasploit (exploitation/escalation), John the Ripper (cracking).
  • Disable LM/NTLM and add MFA to defend against credential dumping.
decision rules
  • Scenario uses sqlmap to breach a database via input → injection
  • Scenario runs Mimikatz to pull hashes from memory → credential dumping
  • Scenario exploits a misconfig to gain root → vertical privilege escalation
  • Scenario cracks /etc/shadow with John the Ripper → credential dumping/password cracking

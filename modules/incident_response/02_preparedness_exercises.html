Preparedness Exercises

Rehearsing the response plan before a real incident — from low-stakes discussion to full-scale drills on the live network.

overview

Preparedness exercises evaluate and improve readiness to handle incidents. Four types rise in intensity: tabletop (discussion only), walkthrough (hands-on practice of procedures), parallel (simulated incident in a staging environment that mirrors production), and simulation (a realistic attack run on the live production network). The dividing line that matters most: parallel runs in staging, simulation runs in production.

type 01

Tabletop

Discussion-based. Team talks through a scenario and their roles, no actions taken, no time pressure, no system interaction.

type 02

Walkthrough

Hands-on but still off production. Members rehearse specific procedures (comms channels, access points). Good for training new staff.

type 03

Parallel

Runs a simulated incident in a staging environment that mirrors production. Realistic and hands-on without touching live operations.

type 04

Simulation

Realistic scenario on the live production network, often red team vs blue team in real time. Highest fidelity, needs careful safeguards.

the four types

Tabletop
Discussion-based session where the team talks through roles and responses for a hypothetical scenario. No actions, no time pressure, no interaction with the production environment. Like a fire-drill assembly where everyone discusses exits without leaving the building. Best for clarifying roles and finding gaps.
Walkthrough
Step-by-step review where members actively practice parts of the IR plan. More hands-on than a tabletop (verifying comms channels, system access, escalation routes), but still does not touch production. Useful for training and refreshing the team.
Parallel
A simulated incident run in a staging environment that mirrors production. Actions are taken, but in a parallel environment so live operations are unaffected. Like rehearsing on a duplicate stage identical to the real one.
Simulation
A high-intensity, realistic scenario carried out on the live production network. Often a red team (attackers) vs blue team (defenders) running in real time. Closest to a real incident, so it requires monitoring and safeguards to avoid harming operations.

how often

A reasonable cadence: walkthroughs at least quarterly, tabletops at least twice a year, and simulations annually. If a yearly simulation is too costly, rotate between a simulation and a parallel test each year.

EXAM TIPS

what to know for the exam

  • Tabletop = discussion only, no actions, no time pressure. Walkthrough = hands-on rehearsal of procedures, still not on production.
  • Parallel = simulated incident in a STAGING environment that mirrors production. Simulation = realistic attack on the LIVE production network.
  • The key parallel-vs-simulation distinction: staging vs production.
  • Simulations often use a red team (attack) / blue team (defend) construct in real time.
  • Rough cadence: walkthrough quarterly, tabletop twice a year, simulation annually (or alternate simulation/parallel).
decision rules
  • Scenario involves the team talking through roles with no systems touched → tabletop
  • Scenario involves rehearsing specific response steps off production → walkthrough
  • Scenario involves a hands-on drill in a staging copy of production → parallel
  • Scenario involves a live red-team/blue-team attack on production → simulation

Roots of Trust

The trusted foundation every secure operation builds on. Three forms (TPM, vTPM, HSM) store keys and anchor trust, each suited to a different environment.

overview

A root of trust is the inherently-trusted foundation for all secure operations: it holds cryptographic keys and enables secure boot, and must be secure by design. Three forms matter: a TPM is a physical chip on the motherboard (a hardware root of trust), a vTPM emulates that functionality in software for virtual machines (a virtual root of trust), and an HSM is a standalone appliance that manages keys across many machines. For the exam, focus on what distinguishes the three.

type 01

TPM

Physical chip on the motherboard. Hardware root of trust. Stores keys, validates boot integrity, works with BitLocker. One per machine.

type 02

vTPM

Software emulation of a TPM for virtual machines. Virtual root of trust. Same functions per VM, managed via VMware or Hyper-V.

type 03

HSM

Standalone appliance or card. Dedicated hardware root of trust managing keys across many machines. Enterprise-scale, tamper-evident.

trusted platform module (TPM)

A TPM is a physical chip installed on the motherboard that acts as a hardware root of trust, located within the system firmware. It securely stores cryptographic keys, digital certificates, and hashed passwords, and performs cryptographic operations. The TPM validates the BIOS, operating system, and other critical components during startup to ensure they have not been tampered with, anchoring secure boot and system integrity.

TPMs generate and manage keys (including RSA keys) using a true random number generator, and contain persistent memory for keys like the endorsement key and storage root key, plus versatile memory for platform configuration registers and attestation identity keys. The exam-level takeaway: a TPM is a hardware root of trust that helps a system boot securely by verifying that the BIOS and firmware have not been modified. It also works with full-disk encryption like BitLocker, using stored keys to keep data safe if a device is lost or stolen.

virtual trusted platform module (vTPM)

A vTPM extends TPM functionality into virtualized environments. Unlike a physical TPM, it is a software implementation that emulates TPM security features inside a virtual machine, providing a virtual root of trust. It handles key generation, encryption, decryption, and signature verification, secures the VM boot process, and validates system integrity, giving each VM the same security posture as a physical device.

vTPMs are managed through platforms like VMware or Microsoft Hyper-V, and let virtual instances maintain individual security postures while benefiting from the same attestation and cryptographic capabilities as physical TPMs. In short: a vTPM provides the same functionality as a TPM, but designed specifically for virtual machines.

hardware security module (HSM)

An HSM is a dedicated hardware root of trust designed to manage and protect cryptographic keys and perform cryptographic operations across multiple machines. Unlike a TPM (embedded on an individual motherboard), an HSM is a standalone appliance or specialized card that delivers high-security key management, encryption, decryption, and digital signatures at scale, across many devices.

HSMs generate and securely store keys such as RSA, Elliptic Curve, and AES keys, and are commonly used to protect master encryption keys across physical servers and virtual machines for centralized key management. Typical business cases include managing PKI key pairs, encrypting PINs for payment card systems, processing TLS connections, and signing large DNS zone files. HSMs are designed to be tamper-evident and help mitigate insider threats, but they are expensive due to specialized hardware and difficult to upgrade because they are physical.

EXAM TIPS

what to know for the exam

  • TPM = physical chip on the motherboard. Hardware root of trust. One per machine. Anchors secure boot; works with BitLocker.
  • vTPM = software emulation of a TPM for virtual machines. Same functions, virtual root of trust. Managed by VMware/Hyper-V.
  • HSM = standalone appliance/card managing keys across many machines. Enterprise-scale, tamper-evident, expensive, hard to upgrade.
  • The exam tests the distinction: chip (TPM) vs VM emulation (vTPM) vs standalone multi-machine device (HSM).
  • HSM use cases: PKI key pairs, payment-card PIN encryption, TLS processing, DNS zone signing.
decision rules
  • Scenario involves a chip on the motherboard securing the boot process → TPM
  • Scenario involves giving each virtual machine its own root of trust → vTPM
  • Scenario involves one device managing encryption keys for many servers → HSM
  • Scenario involves a standalone tamper-evident appliance for PKI or payment keys → HSM
  • Scenario pairs full-disk encryption (BitLocker) with a hardware key store → TPM

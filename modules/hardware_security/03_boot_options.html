Boot Options

Verifying integrity before the OS loads. Secure Boot blocks untrusted code from running; Measured Boot records what loaded so tampering can be detected after the fact.

overview

Boot options determine how a system starts and verifies its integrity before the operating system loads. Two work together with UEFI: Secure Boot ensures each component (firmware, OS loader, drivers) has a valid digital signature and blocks anything that fails, actively preventing unauthorized code. Measured Boot records cryptographic hashes of each component into the TPM to create a verifiable log, used for attestation to detect tampering. Secure Boot prevents; Measured Boot detects.

option 01

Secure Boot

Validates the digital signature of every boot component and blocks any that fail. Actively prevents unauthorized code from running. UEFI setting.

option 02

Measured Boot

Records cryptographic hashes of each component into the TPM as a verifiable log. Detects tampering for attestation, but does not block or halt.

UEFI vs BIOS

Secure Boot is enabled in the Unified Extensible Firmware Interface (UEFI) settings. UEFI is modern system firmware that replaces the older BIOS, adding a graphical interface, mouse support during boot, and improved security. It also supports disks larger than 2 TB, provides a CPU-independent architecture, and allows a pre-OS environment that can include networking and even web browsing.

secure boot

When enabled, Secure Boot performs three verifications during startup to ensure the system has not been compromised:

  1. Firmware checks the integrity of the UEFI executable files and the operating system loader to confirm they have not been tampered with.
  2. Windows boot components verify the digital signature of each component before loading, blocking any that fail and triggering an alert.
  3. Boot-critical drivers are validated against their known-good hashes; only verified drivers are allowed to load.

The result is that the OS starts with only trusted, verified components, significantly reducing the risk of boot-level attacks. A key limitation: Secure Boot only functions with Windows, so it cannot be used with a dual-boot system or Linux.

measured boot

Measured Boot (sometimes called Trusted Boot) checks the integrity of software components during startup using cryptographic hashing, comparing each component against a known-good catalog of expected hash values. The device's TPM records the hashes of key boot stages (UEFI firmware, boot loader, OS kernel, critical drivers) into the TPM, building a verifiable log of the boot sequence.

Crucially, Measured Boot does not halt the boot process even when it detects a discrepancy, it records what happened. Those logs can be accessed by remote management systems, letting security teams verify integrity against known-good states and identify unauthorized modifications. The slight cost is a small increase in boot time.

EXAM TIPS

what to know for the exam

  • Secure Boot PREVENTS: it validates digital signatures and blocks unsigned/tampered components. Measured Boot DETECTS: it records hashes to the TPM but does not block.
  • Measured Boot stores its measurements in the TPM and is used for attestation. Secure Boot relies on UEFI digital signatures.
  • Secure Boot is a UEFI feature and only works with Windows, so no dual-boot or Linux.
  • UEFI replaces BIOS: GUI, mouse support, >2 TB disks, CPU-independent, pre-OS environment.
  • If a question stresses "blocks untrusted code" → Secure Boot. If it stresses "creates a log to detect tampering" → Measured Boot.
decision rules
  • Scenario involves blocking unsigned drivers or boot loaders from running → Secure Boot
  • Scenario involves recording a hash log of the boot sequence for later analysis → Measured Boot
  • Scenario involves attestation of boot integrity to a remote system → Measured Boot
  • Scenario notes the boot process is not halted despite detecting a change → Measured Boot
  • Scenario involves enabling a firmware feature that requires Windows and breaks Linux dual-boot → Secure Boot

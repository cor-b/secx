Security Coprocessors

Specialized hardware that carves out a protected space for sensitive operations, keeping keys and critical processes safe even if the main OS is compromised.

overview

Security coprocessors are specialized hardware components that perform cryptographic operations and store sensitive data in a protected environment. Two forms matter: CPU security extensions (Intel TXT, AMD SEV) create a secure execution environment inside the CPU to shield critical processes, and the Secure Enclave is an Apple-specific coprocessor with its own processor and memory dedicated to encryption keys and biometric data. Both keep sensitive operations isolated from the main operating system.

type 01

CPU Security Extensions

Hardware features (Intel TXT, AMD SEV) that build a secure execution environment in the CPU, isolating sensitive operations from the OS and hypervisor.

type 02

Secure Enclave

Apple-specific coprocessor with its own processor and memory, dedicated to encryption keys, biometric data, and secure transactions.

CPU security extensions

CPU security extensions are hardware-based technologies that create secure execution environments within the CPU to protect critical processes and data from unauthorized access and tampering. They are enabled by configuring system firmware and settings. By isolating the parts of the system where sensitive operations occur, they create a trusted execution environment shielded from the main OS and other applications, preventing malware, rootkits, and other malicious software from interfering.

Intel Trusted Execution Technology (TXT)
Validates that the software environment is secure before sensitive applications run. Protects against attacks that aim to alter the execution environment, confirming a trusted state before critical workloads launch.
AMD Secure Encrypted Virtualization (SEV)
Encrypts data inside virtual machines. Even if the hypervisor is compromised, the VMs remain protected from unauthorized access, preserving VM isolation.

These extensions specifically defend against memory corruption, unauthorized access to sensitive data, and breaches of virtual-machine isolation, the kinds of attacks that exploit vulnerabilities in the OS, applications, or hypervisor.

secure enclave

The Secure Enclave is a secure execution environment specific to Apple hardware that operates independently from the main CPU. Unlike general CPU security extensions, it is purpose-built to manage sensitive operations: storing encryption keys, handling biometric data, and performing secure transactions. It has its own dedicated processor and memory, making it highly resistant to unauthorized access even if the main operating system is compromised.

By keeping these operations completely separate from the main CPU and the rest of the system, the Secure Enclave protects against attacks targeting the OS or its applications. Even if the main system is breached, the data handled inside the enclave (cryptographic keys, biometric information) stays protected, significantly reducing the risk of a data breach.

EXAM TIPS

what to know for the exam

  • Security coprocessors create a protected environment for crypto operations and sensitive data, isolated from the main OS.
  • CPU security extensions: Intel TXT and AMD SEV. They build a trusted execution environment in the CPU.
  • AMD SEV encrypts VM memory so VMs stay protected even if the hypervisor is compromised.
  • Secure Enclave = Apple only. Dedicated processor and memory for keys and biometric data. This is the most-tested distinguishing fact.
  • Both share the goal: sensitive operations remain protected even if the main operating system is compromised.
decision rules
  • Scenario names Intel TXT or AMD SEV → CPU security extensions
  • Scenario involves encrypting VM memory so a compromised hypervisor cannot read it → AMD SEV
  • Scenario involves an Apple device isolating biometric data and keys → Secure Enclave
  • Scenario asks for a dedicated processor separate from the main CPU on Apple hardware → Secure Enclave
  • Scenario involves a trusted execution environment built into the CPU → CPU security extensions

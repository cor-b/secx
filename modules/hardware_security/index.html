Hardware Security

Implementing security in the hardware itself: trusted foundations, verified boot, isolated crypto operations, encrypted storage, and resilient, virtualized systems.

overview

Domain 3, Objective 3.4: given a scenario, implement hardware security technologies and techniques. Topics cover roots of trust (TPM/vTPM/HSM), boot integrity (Secure Boot, Measured Boot), security coprocessors, self-encrypting drives, host-based encryption, self-healing hardware, and virtual hardware.

CAPEC

MITRE's free catalog of common attack patterns. Each entry spells out an attack's prerequisites, steps, and outcomes, so developers can design defenses against it early.

overview

CAPEC (Common Attack Pattern Enumeration and Classification) is a MITRE threat modeling framework that categorizes and describes common attack patterns. Each entry details an attack's prerequisites, potential outcomes, and the typical steps involved. It's a free public resource (capec.mitre.org), most useful for anticipating threats during the software development lifecycle.

what CAPEC provides

CAPEC gives a structured way to understand attacks. Each entry goes beyond a general description with detailed attributes:

  • Prerequisites: conditions needed for the attack (specific software configs, user permissions).
  • Potential outcomes: what success looks like (data theft, disruption, denial of service).
  • Steps: the typical sequence attackers follow.

It organizes patterns into categories and subcategories so teams can quickly locate specific threats or related techniques, supporting both specific and broad defense strategies.

use in the software development lifecycle

CAPEC's key strength is the SDLC: security teams and developers reference it early to anticipate which attacks may target their application and build in protections proactively (secure coding, input validation, encryption). Example: a team that finds a potential SQL injection vulnerability consults the relevant CAPEC entry to understand how attackers exploit it, then applies the recommended defenses, parameterized queries and input validation, to secure database interactions before deployment.

EXAM TIPS

what to know for the exam

  • CAPEC = Common Attack Pattern Enumeration and Classification, a MITRE catalog of attack patterns.
  • Each entry details prerequisites, potential outcomes, and typical steps.
  • Free public resource (capec.mitre.org); organized into categories/subcategories.
  • Primary use is the software development lifecycle: anticipate and design against attacks early.
  • Don't confuse with MITRE ATT&CK: CAPEC = attack patterns (app/dev focus); ATT&CK = adversary TTPs.
decision rules
  • Scenario references a catalog of attack patterns with prerequisites/steps → CAPEC
  • Scenario uses it during development to design secure code → CAPEC in the SDLC
  • Scenario looks up SQL injection defenses by attack pattern → CAPEC entry
  • Scenario maps adversary TTPs across attack stages → MITRE ATT&CK (not CAPEC)

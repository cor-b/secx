STRIDE

An acronym that turns threat modeling into a checklist: six categories of threat to walk through during design, each with a matched mitigation.

overview

STRIDE identifies and categorizes threats across six types: Spoofing, Tampering, Repudiation, Information disclosure, Denial of service, and Elevation of privilege. It's commonly used during the design phase of software development to evaluate how each threat could be exploited.

the six categories

Spoofing
Impersonating another user or system to gain unauthorized access. Mitigate with multi-factor authentication.
Tampering
Unauthorized modification of data or processes. Mitigate with integrity checks and input validation.
Repudiation
Denying actions after performing them, challenging accountability. Mitigate with secure logging/audit trails.
Information disclosure
Unauthorized release of sensitive data (leaks, breaches). Mitigate with encryption (in transit and at rest).
Denial of service
Disrupting system availability so legitimate users can't access it. Mitigate with rate limiting.
Elevation of privilege
Gaining higher access rights than intended. Mitigate with role-based access controls.

using STRIDE in design

Walk each category against the system being designed: for spoofing, add MFA; for tampering, add integrity checks where input validation is weak; for repudiation, add secure logging; for information disclosure, encrypt data in transit and at rest; for denial of service, add rate limiting; for elevation of privilege, enforce RBAC. Applying STRIDE during design proactively identifies and addresses each threat category, building security in from the start.

EXAM TIPS

what to know for the exam

  • STRIDE = Spoofing, Tampering, Repudiation, Information disclosure, Denial of service, Elevation of privilege.
  • Used during the design phase to categorize and address threats proactively.
  • Match mitigations: spoofing→MFA, tampering→integrity checks, repudiation→logging, info disclosure→encryption, DoS→rate limiting, EoP→RBAC.
  • Repudiation is about deniability/accountability; non-repudiation (logging) counters it.
  • STRIDE is design-focused, unlike kill chain/ATT&CK which model live attacks.
decision rules
  • Scenario impersonates a legitimate user → spoofing
  • Scenario denies having performed an action → repudiation
  • Scenario leaks sensitive data → information disclosure
  • Scenario gains admin rights it shouldn't have → elevation of privilege

Diamond Model of Intrusion Analysis

Dissecting an intrusion into four interconnected points, adversary, infrastructure, capability, victim, plus meta-features for context. Trace any vertex to reach the others.

overview

The Diamond Model dissects cyber incidents using four interconnected components visualized as a diamond's vertices: adversary (who, their motives/resources), infrastructure (tools, networks, C2 servers, phishing sites), capability (the TTPs used), and victim (the target and exploited vulnerabilities). Meta-features add context.

vertex 01

Adversary

The individual/group behind the attack: motives, resources, intent.

vertex 02

Infrastructure

Tools, networks, resources used: C2 servers, malware, phishing sites.

vertex 03

Capability

The TTPs employed: exploiting a vulnerability, deploying malware.

vertex 04

Victim

The target (org, individual, system) and the vulnerabilities exploited.

the four components and meta-features

The four vertices are interconnected, each influencing the others: an adversary uses their infrastructure to deploy a capability against a victim. Alongside them, meta-features add context and depth:

Timestamp
When the attack occurred.
Phase
The stage of the attack.
Result
The outcome (successful access, data theft).
Direction
The network path/connection used.
Methodology
The attack techniques/tactics used.
Resources
The assets the adversary needed (tools, exploits).

applying the model

Investigation example: malware on a victim system → analyze the capability (it connects to a malicious C2 domain) → trace the infrastructure (the C2 server) → attribute to an adversary (linking the C2 IP to a known group). Proactively, defenders break the attack at multiple points: filter email/train users (victim), block C2 IPs on the firewall (infrastructure), or add a DNS rule redirecting malicious domains. The Diamond Model can integrate with MITRE ATT&CK to map specific tactics/techniques to the broader intrusion strategy.

EXAM TIPS

what to know for the exam

  • Four vertices: adversary, infrastructure, capability, victim, interconnected as a diamond.
  • Adversary = who; infrastructure = tools/networks/C2; capability = TTPs; victim = target + vulnerabilities.
  • Meta-features: timestamp, phase, result, direction, methodology, resources.
  • Tracing one vertex (e.g. capability → infrastructure → adversary) supports attribution.
  • Integrates with MITRE ATT&CK to map tactics/techniques to the intrusion.
decision rules
  • Scenario analyzes an intrusion via adversary/infrastructure/capability/victim → Diamond Model
  • Scenario blocks the attacker's C2 servers → infrastructure vertex
  • Scenario attributes an attack to a known group → adversary vertex
  • Scenario records when and how the attack happened → meta-features (timestamp, methodology)

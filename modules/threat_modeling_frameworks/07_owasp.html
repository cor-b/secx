OWASP

A nonprofit's toolkit for web-app security: the Top 10 risk checklist, the ASVS standard for detailed requirements, and ZAP for dynamic testing.

overview

OWASP (Open Web Application Security Project) is a nonprofit providing resources to improve software security, especially web applications. Three key offerings: the OWASP Top 10 (most critical web app risks), the ASVS (Application Security Verification Standard, detailed requirements), and ZAP (Zed Attack Proxy, a dynamic testing tool).

resource 01

OWASP Top 10

The 10 most critical web app risks (injection, broken auth, XSS). A health checklist/baseline for compliance.

resource 02

ASVS

Application Security Verification Standard: detailed, specific security controls (session mgmt, validation, auth) for the design phase.

ZAP

Zed Attack Proxy: dynamic application security testing tool that scans for SQLi, XSS, etc., automated and manual.

OWASP Top 10

The OWASP Top 10 lists the most critical web application security risks (injection flaws, broken authentication, cross-site scripting). It's a health checklist for web apps: developers assess "vitals" like validating inputs (injection) and reviewing authentication. Enterprises often use it as a baseline to meet regulatory/compliance requirements and ensure apps aren't vulnerable to common, preventable attacks.

ASVS and ZAP

The ASVS (Application Security Verification Standard) goes beyond the Top 10's general risks, providing specific, detailed security requirements (secure session management, data validation, authentication controls). Teams use it in design and development to set clear security objectives and verify their app meets them.

ZAP (Zed Attack Proxy) is a dynamic application security testing tool that probes a running application for weaknesses like SQL injection and XSS, like inspecting a house for cracks before buying. It automates many checks and supports manual exploration, suitable for routine scans or deep testing. In practice the three combine: use the Top 10 as a starting checklist, apply ASVS requirements to build robust features, and run ZAP to catch hidden vulnerabilities before deployment.

EXAM TIPS

what to know for the exam

  • OWASP = nonprofit focused on web application security.
  • Top 10 = the most critical web app risks (a checklist/baseline); ASVS = detailed, specific security requirements.
  • ZAP = Zed Attack Proxy, a dynamic (running-app) security testing tool.
  • Workflow: Top 10 checklist → ASVS requirements → ZAP testing.
  • Top 10 is general/awareness; ASVS is granular/verifiable.
decision rules
  • Scenario uses a checklist of the top web app risks → OWASP Top 10
  • Scenario needs detailed, verifiable security requirements → ASVS
  • Scenario dynamically scans a running app for SQLi/XSS → ZAP
  • Scenario sets a compliance baseline for web apps → OWASP Top 10

Application Management

Controlling what software can run, and containing the riskiest activity (web browsing) away from the rest of the system.

overview

Application management oversees and controls applications so they do not introduce vulnerabilities or compromise system integrity. Two concepts: application control uses policies and tools (allow lists and block lists) to restrict which applications can run, and browser isolation separates web-browsing activity from the rest of the system, containing web-based threats in an isolated environment.

concept 01

Application Control

Policies and tools that decide which apps can run, using allow lists and block lists. Prevents unauthorized or malicious software from executing.

concept 02

Browser Isolation

Separates browsing from the OS by running the browser in a VM, remote server, or container, so web threats stay contained.

application control

Application control decides which applications can run on a system, preventing unauthorized or malicious software from compromising the network. It uses two list types (formerly called whitelists and blacklists):

  • Allow lists: specify the applications that are permitted to run
  • Block lists: specify applications that are not permitted (for example games or unapproved software)

Application control is commonly implemented through Windows Group Policy, where administrators define rules and push allow/block list policies to all workstations in the domain, preventing unapproved software from being installed. More advanced solutions, such as a HIPS, can block not just installation but the execution of certain files, like portable executables that run without being installed, stopping harmful software before it ever runs.

browser isolation

Browser isolation separates web-browsing activity from the rest of the system, preventing web-based threats from spreading. It runs the browser in an isolated environment, a virtual machine, remote server, or secure container, creating a barrier between dangerous websites and the main OS. Malicious content like drive-by downloads, malware, or harmful scripts stays contained within the isolated environment.

In the enterprise, browser isolation can be cloud-based or on-premises, integrated with secure web gateways or firewalls. Administrators route web traffic through isolated environments by policy: high-risk sites or specific user groups can be auto-isolated, and policies can enforce read-only browsing or block downloads entirely. Centralized consoles let security teams monitor browsing, adjust isolation rules, and apply dynamic isolation based on real-time site reputation or user behavior.

EXAM TIPS

what to know for the exam

  • Application control = decide which apps run, via allow lists (permitted) and block lists (denied). Formerly whitelists/blacklists.
  • Windows Group Policy pushes allow/block rules to all domain workstations. A HIPS can block execution of portable executables, not just installation.
  • Browser isolation runs the browser in a VM, container, or remote server so web threats never touch the host OS.
  • Isolation can be cloud-based or on-premises, and can enforce read-only browsing or block downloads.
  • Application control governs local software; browser isolation governs web activity, they are complementary.
decision rules
  • Scenario involves permitting only approved applications to run → application control (allow list)
  • Scenario involves blocking games or unapproved software → application control (block list)
  • Scenario involves stopping a portable executable from running → application control via HIPS
  • Scenario involves containing risky web browsing away from the OS → browser isolation
  • Scenario involves running the browser in a remote container with downloads blocked → browser isolation

Threat Protection

The host-level defensive stack: anti-malware to catch malicious software, a host firewall to filter traffic, and HIDS/HIPS to detect (and block) suspicious behavior.

overview

Threat protection is the set of measures that detect, prevent, and respond to attacks on a host. Four components work together: anti-malware finds and removes malicious software, a host-based firewall filters traffic to and from the endpoint, a HIDS watches system activity and alerts, and a HIPS goes further by actively blocking attacks. The HIDS-vs-HIPS distinction (detect vs detect-and-block) is the key exam point.

control 01

Anti-malware

Detects, prevents, and removes malicious software using signatures, behavior analysis, and heuristics. Microsoft Defender Antivirus combines AV and anti-spyware.

control 02

Host-Based Firewall

Software on the individual endpoint that accepts or drops packets by application, port, or IP. Targets inbound traffic. Windows Firewall, iptables, OS X firewall.

control 03

HIDS

Host intrusion detection: analyzes logs, configs, and behavior to spot suspicious activity, then alerts and logs. Detection only.

control 04

HIPS

Host intrusion prevention: detects via signatures and anomalies, then actively blocks the attack. Often tied to a central orchestrator.

anti-malware

Anti-malware detects, prevents, and removes malicious software (viruses, worms, ransomware, adware, spyware) using a mix of signature detection, behavior analysis, and heuristics. It also provides valuable logging of infection attempts. Historically antivirus and anti-spyware were separate tools; today they are usually combined, for example Microsoft Defender Antivirus folds both into one platform. Keeping definitions current and scanning regularly remains essential.

Because email is a major malware vector, supporting controls matter: disabling automatic image previews, spam filters (often built into anti-malware) to block malicious mail, user training against phishing links, and browser extensions that warn about malicious sites. Anti-malware can also detect spyware such as keyloggers that capture keystrokes.

host-based firewall

Unlike network firewalls, host-based (personal) firewalls are software installed directly on a machine. Most operating systems include one, but it usually must be enabled. They accept or drop packets based on application or port, focused on inbound traffic. Windows Firewall can block by IP or port; Linux uses iptables (more powerful than the older ipchains); macOS provides a firewall in System Preferences.

A properly configured host firewall blocks spoofed and unwanted source ranges. Attackers often spoof private ranges (10.0.0.0, 172.16.0.0, 192.168.0.0), and attacks can originate from loopback (127.0.0.0), multicast (224.0.0.0), and experimental (240.0.0.0) ranges, all good candidates to block at the host.

HIDS vs HIPS

Both run on the individual host. A HIDS (host-based intrusion detection system) monitors closely, analyzing log files, system configuration, and application behavior to spot unauthorized changes, unexpected processes, or unusual access. When it finds something, it generates alerts and logs the details, but it does not stop the activity.

A HIPS (host-based intrusion prevention system) goes a step further: it detects using both signature-based detection (known attack patterns) and anomaly-based detection (deviations from normal behavior), and then actively blocks the action before harm is done. HIPS is often integrated with a centralized management server or orchestrator that aggregates alerts from all endpoints for a unified view.

EXAM TIPS

what to know for the exam

  • Anti-malware uses signatures + behavior analysis + heuristics. Modern suites combine AV and anti-spyware (Microsoft Defender Antivirus).
  • Host-based firewall = software on the endpoint, filters mainly inbound traffic by app/port/IP. Block private (10/172.16/192.168), loopback (127), multicast (224), experimental (240) ranges.
  • Linux host firewall: iptables (newer/more powerful than ipchains).
  • HIDS detects and alerts only. HIPS detects AND blocks. This detect-vs-prevent distinction is the core exam point.
  • HIPS uses signature-based AND anomaly-based detection, often reporting to a central orchestrator.
decision rules
  • Scenario involves detecting and quarantining ransomware on a host → anti-malware
  • Scenario involves software on one machine blocking inbound traffic by port/IP → host-based firewall
  • Scenario involves monitoring a host and alerting on suspicious behavior (no blocking) → HIDS
  • Scenario involves a host tool that blocks a malicious process in real time → HIPS
  • Question contrasts detect-only vs detect-and-block → HIDS vs HIPS

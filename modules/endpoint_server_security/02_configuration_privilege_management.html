Configuration & Privilege Management

Keeping every system consistently configured and giving every user only the access they need. Two disciplines that together enforce a secure, compliant baseline.

overview

Configuration and privilege management are the processes and tools for managing system settings and user permissions so the enterprise operates securely and consistently. Configuration management controls system settings, software versions, and updates (SCCM for Windows, Ansible across platforms). Endpoint privilege management controls access levels on individual devices to enforce least privilege (Linux sudo, Windows LAPS).

Configuration Management

configuration management

Configuration management controls and maintains system settings, software versions, and updates so systems stay secure and behave consistently across the organization. It works by defining rules for how software is installed, updated, and maintained on every device.

SCCM

Microsoft System Center Configuration Manager (SCCM) automates the deployment of software updates, security patches, and configuration settings across all Windows devices, keeping systems current and compliant with security policy.

Ansible and declarative configuration

Ansible is an open-source platform for automating configuration across Linux, Windows, and network devices. It uses human-readable scripts called playbooks and works declaratively: you define the desired end state (a package is installed, a service is running) and Ansible figures out the steps to reach it. This lets administrators describe what the final configuration should be and let the tool handle the details.

endpoint privilege management

Endpoint privilege management controls user permissions on individual devices, reducing risk by restricting access to what each role actually needs. It enforces the principle of least privilege, limiting the chance of unauthorized actions whether accidental or malicious.

Linux: sudo

On Linux, privilege management is commonly implemented with the sudo command, which lets users run specific administrative commands without full admin rights. Editing the /etc/sudoers file defines precise command permissions for users or groups. Advanced configurations add time-based access controls and logging to track and audit privileged activity in real time.

Windows: LAPS

Microsoft Local Administrator Password Solution (LAPS) manages local administrative accounts. It automatically generates a complex, unique password for the local admin account on each endpoint and stores it securely in Active Directory with tightly controlled access. Combined with automated password rotation, this minimizes the risk of lateral movement by attackers exploiting shared or static local-admin credentials, and it provides audit trails of password access.

EXAM TIPS

what to know for the exam

  • Configuration management = consistent settings/software/updates. SCCM (Windows), Ansible (cross-platform, declarative playbooks).
  • Declarative = you specify the desired end state; the tool reaches it. Ansible is the example.
  • Endpoint privilege management = least privilege on each device.
  • Linux: sudo with /etc/sudoers grants per-command admin access without full rights.
  • Windows: LAPS rotates and securely stores unique local-admin passwords in AD, blocking lateral movement via shared credentials.
decision rules
  • Scenario involves pushing consistent settings and patches to all Windows devices → SCCM
  • Scenario involves declarative playbooks defining a desired system state → Ansible
  • Scenario involves letting a user run one admin command without full rights → sudo
  • Scenario involves rotating unique local-admin passwords to stop lateral movement → LAPS
  • Question stresses enforcing least privilege on endpoints → endpoint privilege management

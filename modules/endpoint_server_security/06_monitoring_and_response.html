Monitoring & Response

Turning the constant noise of system activity into signal: deliberate logging, a SIEM to aggregate and correlate it, and EDR to detect and contain threats at the endpoint.

overview

Monitoring and response is the continuous observation of system activity to detect, analyze, and respond to threats. Two pillars: event logging and monitoring records and reviews system and application activity to spot abnormal behavior (centralized and correlated through a SIEM), and EDR gives endpoints real-time visibility, automated detection, and the ability to isolate and quarantine malicious activity fast.

pillar 01

Logging & Monitoring

Records system, app, user, and network events into logs and audit trails. Centralized and correlated by a SIEM for real-time detection and forensics.

pillar 02

EDR

Endpoint detection and response: continuous endpoint visibility, machine-learning detection of unknown threats, and rapid isolation of compromised devices.

logging and monitoring

Logs capture system events, application events, user actions, and network activity. Defenders analyze them to find indicators of attack and compromise and to reconstruct attack timelines. The core challenge: you cannot log everything, each log consumes processing, memory, and storage, so logging too much degrades the systems you protect, while logging too little leaves gaps. This balance demands deliberate decisions about what to log.

audit log management plan

  • Defines what is logged, how long logs are retained, and how they are backed up
  • Sets how often logs are reviewed
  • Protects logs from unauthorized deletion, sometimes requiring two-person approval for any log modification or deletion

thresholds and audit trails

Deciding thresholds matters: log every failed login, or only after the third attempt? Too much floods the system; too little misses a brute-force attack. Logging user additions, modifications, and deletions maintains visibility, since attackers often make such changes after a breach. An audit trail is a detailed record of transactions and activities, noting whether attempts succeeded or failed. Attackers may try to cover their tracks (scrubbing the logs); missing time entries or unexplained gaps are signs of tampering.

SIEM

A Security Information and Event Management (SIEM) platform combines security event management and security information management, and is typically run by a Security Operations Center (SOC). Tools include ArcSight, AlienVault, and QRadar. Two core strengths:

  • Data aggregation: collects logs from many sources (DHCP, DNS, routers, firewalls, IDS) into one centralized database
  • Data correlation: connects events across sources, normalizing timestamps to a standard like UTC so a timeline can be assembled automatically

SIEMs provide real-time monitoring and automated alerts, and serve as a central repository for regulatory audits and forensic analysis.

endpoint detection and response (EDR)

EDR gives security teams deep visibility over endpoints and detects and responds to threats. Unlike simple anti-malware (which focuses on known threats), EDR continuously monitors processes, file access, and network activity, and notices changes from the device's baseline, so it can identify previously unknown attack patterns and ongoing attacks.

  • Isolation: on detecting suspicious behavior, EDR can immediately disconnect the device from the network while still allowing remote access for investigation, stopping malware or data exfiltration from spreading
  • Machine learning: ML models learn normal behavior and flag subtle anomalies (unfamiliar connections, unauthorized scripts) that traditional tools miss, even threats never seen before
  • Forensics: historical data shows how an attack started, which files were affected, and how it spread, e.g. a ransomware timeline from download to encryption
  • Remediation: after isolation, teams analyze the device, find the entry point, eradicate the threat, then safely reconnect

EXAM TIPS

what to know for the exam

  • You cannot log everything: balance coverage against performance and storage. An audit log management plan defines what/retention/backup/review.
  • Protect log integrity (e.g. two-person approval for deletion). "Scrubbing the logs" / missing time gaps = tampering.
  • SIEM = SEM + SIM, run by a SOC. Core functions: data aggregation (collect) and data correlation (connect, normalize to UTC). Tools: ArcSight, AlienVault, QRadar.
  • EDR monitors endpoints in real time, uses machine learning to catch unknown threats, and can isolate/quarantine a compromised device.
  • EDR vs anti-malware: anti-malware blocks known threats; EDR detects unknown patterns and responds/contains.
decision rules
  • Scenario involves aggregating and correlating logs from many devices in one platform → SIEM
  • Scenario involves normalizing timestamps to UTC to build an attack timeline → SIEM data correlation
  • Scenario involves isolating a compromised laptop while keeping it reachable for investigation → EDR
  • Scenario involves machine learning catching a never-before-seen endpoint threat → EDR
  • Scenario involves deciding to log failed logins only after the third attempt → logging threshold

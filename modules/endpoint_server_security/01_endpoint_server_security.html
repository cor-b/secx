Endpoint and Server Security

Hardening the machines that run the enterprise: managing configuration and privilege, locking down the OS, protecting against threats, and watching everything continuously.

overview

Domain 3, Objective 3.2: given a scenario, analyze requirements to enhance the security of endpoints and servers. A secure enterprise depends on systems being correctly configured, tightly permissioned, monitored, and defended. This section covers configuration and privilege management, operating system security, threat protection, application management, monitoring and response, mobile management, and attack surface management.

theme 01

Configure & Restrict

Configuration management (SCCM, Ansible) and endpoint privilege management (sudo, LAPS) keep systems consistent and enforce least privilege.

theme 02

Harden & Defend

OS security (SELinux/MAC), threat protection (anti-malware, host firewall, HIDS/HIPS), and application control keep the system itself secure.

theme 03

Watch & Respond

Logging, SIEM, and EDR provide visibility, detection, and rapid isolation of compromised endpoints.

theme 04

Extend & Reduce

Mobile device management secures phones and tablets; attack surface management finds and shrinks every avenue of attack.

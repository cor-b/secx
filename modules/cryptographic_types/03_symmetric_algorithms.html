Symmetric Algorithms

Two ways to apply a symmetric key: stream ciphers (bit by bit, for real-time data) and block ciphers (fixed chunks, for files), plus the named algorithms that use each.

overview

Symmetric algorithms split into stream ciphers (encrypt bit/byte at a time using a keystream XOR'd with plaintext, ideal for real-time audio/video) and block ciphers (encrypt fixed-size blocks using confusion and diffusion, ideal for files). Knowing which algorithm is which type, and which are deprecated, is the core of this lesson: AES, DES/3DES, Blowfish/Twofish, RC4/ChaCha20.

type 01

Stream Cipher

Encrypts continuously, bit/byte at a time, via a keystream XOR'd with plaintext. Needs a unique IV. Real-time use (audio/video). RC4, ChaCha20.

type 02

Block Cipher

Encrypts fixed-size blocks (64/128-bit) using confusion + diffusion across multiple rounds. Modes: ECB, CBC, GCM. DES, 3DES, AES, Blowfish, Twofish.

stream vs block ciphers

Stream ciphers generate a pseudo-random keystream from the secret key and XOR it with the plaintext, encrypting continuously, which suits real-time data of unknown length. Each keystream must be unique, ensured by an initialization vector (IV); a weak or predictable IV breaks the encryption (WEP's short 24-bit IV is the classic failure).

Block ciphers encrypt fixed-size blocks (usually 64 or 128 bits) using confusion (making the key-ciphertext relationship complex) and diffusion (spreading plaintext influence across the ciphertext so a tiny input change alters the whole output), applied over multiple rounds. Their mode of operation defines how blocks interact: ECB (Electronic Code Book), CBC (Cipher Block Chaining), and GCM (Galois/Counter Mode).

the algorithms

AES
Advanced Encryption Standard, the current standard. Block (128-bit) or stream (via GCM). Key sizes 128/192/256-bit. Used in WPA2/WPA3 Wi-Fi, file transfers, messaging apps; hardware-accelerated on modern CPUs. Being usable as both block and stream cipher is distinctive to AES.
DES / 3DES
DES: early standard, 56-bit key, 64-bit blocks, now insecure to brute force. 3DES: applies DES three times (~112-bit effective). 3DES is more secure but slower; used in banking/legacy systems, largely replaced by AES.
Blowfish / Twofish
Blowfish: fast, variable key (32–448-bit), but a 64-bit block makes it risky for large data. Twofish: successor, 128-bit blocks, up to 256-bit keys. 64-bit-and-below block sizes are vulnerable to collision attacks on large datasets, hence Twofish.
RC4 / ChaCha20
RC4: pure stream cipher, very fast, but insecure (weak initial keystream); used in old SSL/WEP. ChaCha20: secure modern replacement. ChaCha20 is used in TLS and performs well on both servers and low-power devices.

EXAM TIPS

what to know for the exam

  • Stream ciphers encrypt bit/byte at a time (real-time data); block ciphers encrypt fixed blocks (files). RC4/ChaCha20 = stream; DES/3DES/Blowfish/Twofish = block; AES = both.
  • Stream ciphers need a unique, unpredictable IV. WEP's weak 24-bit IV is the classic vulnerability.
  • Block ciphers rely on confusion (complex key-ciphertext link) and diffusion (small change → big output change). Modes: ECB, CBC, GCM.
  • DES (56-bit) and RC4 are insecure. 3DES is slow/legacy. AES and ChaCha20/Twofish are current.
  • Block sizes of 64-bit and below (Blowfish) are vulnerable to collision attacks on large data.
decision rules
  • Scenario involves encrypting a live audio/video stream → stream cipher (ChaCha20)
  • Scenario involves encrypting files in fixed chunks → block cipher (AES)
  • Scenario names a weak/short IV enabling attack → stream cipher IV weakness (WEP/RC4)
  • Scenario asks the current symmetric standard → AES
  • Scenario describes applying DES three times → 3DES

Cryptographic Types

The two families of encryption: symmetric (one shared key, fast and bulk) and asymmetric (a key pair for exchange, signatures, and trust).

overview

Domain 3, Objective 3.8: given a scenario, apply an appropriate cryptographic use case and/or technique. Symmetric cryptography uses one key for both encrypt and decrypt, fast and ideal for bulk data at rest. Asymmetric (public-key) cryptography uses a key pair for secure key exchange, digital signatures, and authenticity, at higher computational cost. Real systems combine both: asymmetric to exchange a key, symmetric to encrypt the bulk traffic.

family 01

Symmetric

Same key encrypts and decrypts. Fast, low-resource, great for bulk data. AES is the standard. Challenge: distributing the key securely.

family 02

Asymmetric

Public/private key pair. One key encrypts, the other decrypts. Powers key exchange, digital signatures, and authentication. Slower, small data only.

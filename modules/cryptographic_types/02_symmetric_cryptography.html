Symmetric Cryptography

One key for both directions. Fast enough for bulk data, with two notable special cases: the unbreakable one-time pad and resource-frugal lightweight crypto.

overview

Symmetric cryptography uses the same key to encrypt and decrypt, which makes it efficient for securing data quickly and in bulk (the workhorse algorithm is AES). Two special techniques matter: the one-time pad, which is theoretically unbreakable when used correctly, and lightweight cryptography, built for low-power devices like IoT.

core

Symmetric Encryption

Same key both ways. Efficient bulk encryption. Algorithms: AES, DES, 3DES, Blowfish, Twofish, RC4, ChaCha20.

One-Time Pad

Random key as long as the message, XOR'd in. Theoretically unbreakable, but the key must be truly random and never reused.

Lightweight Crypto

Optimized for limited power/resources (IoT, sensors, embedded). Smaller keys, fewer rounds, simpler operations.

symmetric encryption

The same secret key encrypts and decrypts the data, the core advantage being efficient bulk encryption for file storage and network traffic. Common algorithms: AES (strong, fast, works as block or stream cipher), DES (old, 56-bit, now insecure), 3DES (DES applied three times, more secure but slower), Blowfish (fast, variable key) and its successor Twofish (up to 256-bit keys), and RC4 (fast stream cipher, now outdated) with ChaCha20 as its secure replacement.

one-time pad

The one-time pad uses a completely random key that is the exact same length as the message. Each bit of the message is combined with the corresponding key bit via XOR, producing ciphertext with no exploitable patterns, which makes it theoretically unbreakable. Its rules are strict:

  • The key must be truly random, any predictability weakens it
  • The key must never be reused, reuse lets patterns emerge
  • The key must be as long as the message

The catch is key distribution: securely sharing a key as long as the message, used only once, is a major logistical problem, so one-time pads are reserved for the most sensitive (diplomatic, military) communications rather than everyday use.

lightweight cryptography

Lightweight cryptography is designed for resource-constrained environments, IoT devices, smart sensors, embedded systems, where full algorithms like AES can be too computationally heavy. It uses smaller keys, fewer encryption rounds, simpler operations, and shorter block sizes to keep processing and memory demands low. It provides an adequate (not maximum) level of security while letting low-power devices operate effectively.

EXAM TIPS

what to know for the exam

  • Symmetric = same key encrypts and decrypts. Fast, efficient, ideal for bulk data. AES is the standard.
  • One-time pad = random key as long as the message, XOR'd. Theoretically unbreakable IF the key is truly random and never reused. Key distribution is its downfall.
  • Lightweight cryptography targets low-power/IoT devices: smaller keys, fewer rounds, less resource use.
  • ChaCha20 is the modern secure replacement for the deprecated RC4.
decision rules
  • Scenario needs fast bulk encryption with one shared key → symmetric (AES)
  • Scenario describes a random key as long as the message, used once → one-time pad
  • Scenario involves securing an IoT sensor with minimal resources → lightweight cryptography
  • Scenario needs to replace RC4 → ChaCha20

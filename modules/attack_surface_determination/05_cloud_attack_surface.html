Cloud Attack Surface

Entry points in the cloud: the assets meant to stay internal, the ones deliberately public, every service you're running, and the malicious background processes hiding among them.

overview

The cloud attack surface is the vulnerabilities and entry points in cloud infrastructure. Four concepts: internally facing assets (cloud resources accessible only within the network), externally facing assets (publicly accessible cloud resources), cloud services discovery (cataloging all services to find misconfigs), and malicious cloud services/daemons (harmful background processes).

concept 01

Internally Facing

Internal-only resources (databases, VMs, internal APIs). Misconfig can expose them (2023 No-Fly List leak). Fix: AWS Config, Azure Policy.

concept 02

Externally Facing

Public resources (web apps, public APIs, storage buckets, CDNs, DNS). Prime targets. Scan with Shodan/Censys/AWS Inspector.

concept 03

Cloud Services Discovery

Catalog all cloud services to find misconfigs across multi-cloud. Tools: Prisma Cloud, Lacework, CloudTrail (CSPM).

concept 04

Malicious Services/Daemons

Harmful background processes (backdoors, exfiltration) masquerading as legit. Detect with CrowdStrike, GuardDuty, Azure Security Center.

internally vs externally facing assets

Internally facing assets are cloud resources accessible only within the network (databases, VMs, app servers, internal APIs); if misconfigured, they can unintentionally become public. The 2023 US No-Fly List leak exemplifies this, a CommuteAir internal server left unprotected exposed a copy of the list. Fix with config-compliance tools (AWS Config, Azure Policy) and audits. Externally facing assets are intentionally public (web apps, public APIs, storage buckets, CDNs, DNS servers); they're prime targets, e.g. a misconfigured public API with weak auth exposing customer data. Scan with Shodan, Censys, AWS Inspector, Google Cloud Security Scanner; enforce strict access controls (OAuth) and patching.

cloud services discovery and malicious daemons

Cloud services discovery catalogs all cloud services in use (storage, compute, networking, software) across multi-cloud to find vulnerabilities and misconfigurations, undiscovered services can expose data. Use cloud security posture management (CSPM) tools (Prisma Cloud, Lacework, AWS CloudTrail) to automate inventory and monitor configs. Malicious cloud services and daemons are unauthorized background processes enabling unauthorized access, persistent backdoors, or harmful actions, often masquerading as legitimate processes and evading standard monitoring (e.g. a malicious daemon periodically exfiltrating data). Detect with advanced threat detection (CrowdStrike, AWS GuardDuty, Azure Security Center), forensic investigations, and EDR.

EXAM TIPS

what to know for the exam

  • Four cloud concepts: internally facing assets, externally facing assets, cloud services discovery, malicious services/daemons.
  • Misconfigured internal assets can leak publicly (2023 US No-Fly List); fix with AWS Config / Azure Policy.
  • External assets (web apps, public APIs, buckets, CDNs, DNS) are prime targets; scan with Shodan/Censys/Inspector.
  • Cloud services discovery = CSPM tools (Prisma Cloud, Lacework, CloudTrail) catalog services and find misconfigs.
  • Malicious daemons masquerade as legitimate; detect with CrowdStrike, GuardDuty, Azure Security Center.
decision rules
  • Scenario leaks an internal-only database publicly via misconfig → internally facing asset exposure
  • Scenario exposes a public API with weak auth → externally facing asset
  • Scenario catalogs all multi-cloud services for misconfigs → cloud services discovery (CSPM)
  • Scenario finds a hidden process exfiltrating data → malicious cloud daemon

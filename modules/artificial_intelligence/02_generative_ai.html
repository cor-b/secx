Generative AI

AI that creates new content by learning patterns from data. Built on GPT models that break language into tokens and weigh context with attention, with two key security-relevant uses: code assist and documentation.

overview

Generative AI creates new content (text, images, code, music) by learning patterns from existing data. Models like the GPT (generative pre-trained transformer) are trained on large data sets to understand language, context, and structure. Two baseline concepts: code assist (suggesting/completing code) and documentation (auto-generating technical docs).

concept 01

Content Creation

GPT breaks language into tokens, converts them to numbers, and uses attention layers to weigh context (bank = river vs finance).

concept 02

Code Assist

Real-time code suggestions/completion (GitHub Copilot, Tabnine). Faster, more consistent, fewer errors.

concept 03

Documentation

Auto-generate user guides, release notes from the codebase (GitBook, Document360). Stays current with code changes.

how generative AI works

Generative AI studies patterns in existing data to produce outputs that mimic or extend it. A GPT trained on massive text data sets learns word use, relationships, and meanings. It breaks language into tokens (full words, parts of words, or characters), converts them to numerical representations, and processes them with attention mechanisms, layers that focus on specific parts of a passage, giving more weight to words that provide context. In "I went to the bank to fish," attention weights "fish" heavily to interpret "bank" as a riverbank, not a financial institution. This dynamic, context-aware interpretation lets it generate coherent, relevant content with consistency and style.

code assist

Code assist enhances software development by analyzing existing code and patterns to suggest improvements or auto-complete code as the developer types. Tools like GitHub Copilot and Tabnine reduce manual coding and error-correction time, offer everything from syntax fixes to complex snippets aligned with best practices, help less experienced developers learn, and enforce coding standards for consistency, all improving workflow efficiency and code quality.

AI-generated documentation

AI-generated documentation auto-creates technical documents from the codebase or system configuration. Tools like GitBook and Document360 analyze code structure, functions, and dependencies to produce user guides and release notes. This relieves the time-consuming manual burden, keeps docs accurate and aligned with the latest code (auto-updating release notes when features change), and uses natural-language generation to explain functionality for technical and non-technical audiences.

EXAM TIPS

what to know for the exam

  • Generative AI creates new content by learning patterns from existing data; GPT is the canonical model.
  • GPT breaks language into tokens, then uses attention mechanisms to weigh context-relevant words.
  • Code assist = real-time code suggestion/completion (GitHub Copilot, Tabnine).
  • AI-generated documentation auto-produces and updates technical docs (GitBook, Document360).
  • Both code assist and documentation are the two named baseline generative-AI concepts for the exam.
decision rules
  • Scenario suggests/completes code as you type → code assist
  • Scenario auto-writes user guides from the codebase → AI-generated documentation
  • Scenario weighs context to disambiguate a word → attention mechanism
  • Scenario splits text into processable units → tokens

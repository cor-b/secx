Cloud Security Alliance

A framework of guidelines and best practices for securing cloud environments. The CSA STAR program provides a structured certification path for cloud service providers at two levels of rigor.

overview

The Cloud Security Alliance (CSA) fills a gap that general frameworks like ISO 27001 and NIST CSF leave open: they do not address the specific risks of cloud computing environments. CSA provides cloud-specific guidance and, through its STAR program, gives cloud providers a way to demonstrate the quality of their security controls to prospective customers.

The STAR program's publicly accessible registry means any organization can compare cloud providers against a standardized control set before committing to a vendor — making it a procurement and due diligence tool as much as a certification.

three components

01 CSA Framework Guidelines for securing cloud environments: data protection, security management, and compliance across cloud services.
02 Cloud Controls Matrix A standardized set of cloud security controls used as the baseline for STAR assessments and provider comparisons.
03 CSA STAR Program Two-level certification: Level 1 self-assessment, Level 2 third-party audit. Public registry lets customers verify provider security.

CSA framework

The CSA framework is a comprehensive set of guidelines specifically designed for cloud computing environments. It addresses three core areas that general security frameworks underspecify for cloud contexts.

  • Data protection - how data is classified, encrypted, and controlled as it moves between cloud services and geographic regions
  • Security management - identity, access, incident response, and vulnerability management adapted for shared-responsibility cloud models
  • Compliance - mapping cloud controls to regulatory requirements (GDPR, HIPAA, PCI DSS) so organizations can demonstrate compliance even when infrastructure is hosted externally

Cloud Controls Matrix

The Cloud Controls Matrix (CCM) is the control catalog that underpins the STAR program. It is a spreadsheet-style framework of security controls organized by domain, covering areas like identity management, encryption, infrastructure security, and supply chain.

  • Provides a common language for evaluating cloud provider security
  • Maps to other major standards (ISO 27001, NIST CSF, PCI DSS, GDPR) so organizations can assess coverage across multiple frameworks simultaneously
  • Used by providers to self-assess (STAR Level 1) and by auditors to evaluate compliance (STAR Level 2)

CSA STAR program

STAR stands for Security, Trust, Assurance, and Risk. The program gives cloud service providers a structured way to demonstrate their security posture and gives customers a standardized way to evaluate it. Major providers including AWS and Google Cloud participate in STAR.

STAR level 1: self-assessment

The provider voluntarily assesses its own security and privacy controls against the Cloud Controls Matrix or the GDPR Code of Conduct and submits results to the public registry.

  • No independent verification — the provider evaluates itself
  • Results are publicly accessible in the STAR registry, allowing customers to review and compare
  • Low barrier to entry; any cloud provider can participate
  • Example: Dropbox self-assesses and publishes results to assure customers of its practices

STAR level 2: third-party audit

An independent auditor evaluates the provider's controls against the Cloud Controls Matrix. The audit provides external validation rather than self-reported assurance.

  • Significantly more credible than Level 1 because an independent party verifies the claims
  • Similar in concept to SOC 2 Type II — independent verification over a sustained period
  • Results are listed in the STAR registry alongside Level 1 providers, but clearly distinguished
  • Example: Salesforce undergoes third-party audit to offer customers greater confidence

LevelMethodCredibility
STAR Level 1 Self-assessment against CCM or GDPR CoC; results published publicly Lower — no independent verification

exam tips

key takeaways

  • CSA and STAR are specifically for cloud environments. If a question is about securing or certifying cloud service providers, the answer involves CSA or STAR, not ISO 27001 or SOC 2 (even though those also exist in cloud contexts).
  • STAR Level 1 = self-assessment. STAR Level 2 = third-party audit. The distinction maps directly to credibility: Level 2 carries more weight because an independent party verified the controls.
  • The Cloud Controls Matrix (CCM) is the standardized control set used for both levels. Know the name — it appears in questions about comparing cloud providers.
  • The STAR registry is public. Anyone can look up a provider's STAR status. This is the mechanism that makes it a due diligence tool.
  • AWS and Google Cloud participate in STAR — major providers are not exempt just because they are large.
  • STAR Level 2 and SOC 2 Type II are conceptually similar (both use third-party auditors) but STAR is cloud-specific and uses the CCM, while SOC 2 uses the AICPA trust service criteria.
quick decision rules
  • Cloud provider security certification → CSA STAR
  • Self-assessed cloud security, publicly posted → STAR Level 1
  • Third-party audited cloud security → STAR Level 2
  • Standardized control set for comparing cloud providers → Cloud Controls Matrix (CCM)
  • Publicly accessible registry of cloud provider security postures → CSA STAR registry

Vulnerabilities and Attacks

How systems get broken: injection, memory, configuration, and authorization vulnerabilities, plus malicious code, hardware/firmware, memory-based, and electromagnetic attacks.

overview

Domains 3 & 4 OBJ 3.4 / OBJ 4.2: implement hardware security and analyze vulnerabilities/attacks to reduce the attack surface. Topics span injection vulnerabilities (command/code/SQL/XSS/CSRF/SSRF/deserialization), memory-related vulnerabilities (overflows, race conditions, TOCTOU), configuration vulnerabilities (directory misconfig, embedded secrets, EOL software), authorization vulnerabilities (confused deputy, weak ciphers, third parties), malicious code attacks (implants, poisoning), hardware/firmware attacks (BIOS/UEFI, USB), memory-based attacks (buffer overflow, shimming), and electromagnetic attacks (EMI, EMP).

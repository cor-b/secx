Vulnerabilities and Attacks

How systems get broken: flaws in how software is written, configured, and access-managed, and the attacks that exploit them, from injected code to memory manipulation to physical hardware.

overview

Domains 3 and 4, Objectives 3.4 (implement hardware security technologies/techniques) and 4.2 (analyze vulnerabilities and attacks, recommend solutions to reduce the attack surface). Vulnerabilities arise from how software is built, configured, and access-managed; attackers inject code, manipulate memory, or target hardware. The section covers injection, memory-related, configuration, and authorization vulnerabilities, plus malicious code, hardware/firmware, memory-based, and electromagnetic attacks.

area 01

Vulnerabilities

Injection, memory-related, configuration, and authorization weaknesses, where flaws come from.

area 02

Software Attacks

Malicious code (implants, poisoning) and memory-based attacks (buffer overflow, shimming).

area 03

Physical Attacks

Hardware and firmware attacks (BIOS/UEFI, USB) and electromagnetic attacks (EMI, EMP).

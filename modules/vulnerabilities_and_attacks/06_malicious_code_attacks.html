Malicious Code Attacks

Harmful code inserted to disrupt, steal, or manipulate: a hidden persistent backdoor (implant), or corrupting the data a system depends on (poisoning).

overview

Malicious code attacks insert harmful software or code into a system to disrupt, steal, or manipulate data and operations. Two concepts: implants (stealthy persistent backdoors in software/firmware/hardware) and poisoning (tampering with the data or environment a system relies on).

attack 01

Implants

A persistent, stealthy backdoor (software/firmware/hardware) giving ongoing remote access. Common in APT TTPs; can hide undetected for long periods.

attack 02

Poisoning

Tampering with data/environment to alter behavior (corrupting ML training data, malicious software updates). Degrades accuracy subtly.

implants

An implant is a persistent backdoor or unauthorized code inserted into a system for continuous, stealthy access, as software, firmware, or hardware hiding within the system. Implants are dangerous because they may remain undetected for long periods, giving attackers repeated access or control. They're common in advanced persistent threat (APT) tactics. Example: a malicious script hidden in a legitimate application that collects environment variables (including credentials) and exfiltrates them every time a function runs, hard to detect because it's embedded in the app's flow. Mitigate with rigorous code reviews (especially external code), endpoint monitoring (OSSEC, CrowdStrike, Carbon Black), and regular integrity checks on critical files.

poisoning

Poisoning manipulates the data or environment a system relies on to alter its behavior or produce incorrect outcomes, often targeting data-driven systems like machine-learning models or databases. Example: an attacker injects mislabeled spam emails (labeled ham, i.e. legitimate) into a spam filter's training set, degrading its accuracy so spam reaches inboxes. Such subtle manipulations accumulate into significant vulnerabilities. Mitigate by controlling access to training data (trusted sources only), employing data validation to detect abnormal patterns, monitoring model performance for sudden accuracy drops, and retraining with clean, validated data.

EXAM TIPS

what to know for the exam

  • Implant = a persistent, stealthy backdoor (software/firmware/hardware) for ongoing access; hallmark of APTs.
  • Poisoning = tampering with data/environment (e.g. corrupting ML training data, malicious updates) to alter behavior.
  • Implants persist and hide; poisoning degrades accuracy/integrity subtly over time.
  • Detect implants with endpoint monitoring (OSSEC, CrowdStrike, Carbon Black) and integrity checks.
  • Defend poisoning by controlling training-data access, validation, and performance monitoring.
decision rules
  • Scenario hides a persistent backdoor for repeated access → implant
  • Scenario corrupts an ML model's training data → poisoning
  • Scenario injects malicious code into a software update → poisoning
  • Scenario embeds exfiltration in a legitimate app's flow → implant

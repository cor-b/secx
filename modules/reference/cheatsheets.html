boot options

Secure BootMeasured Boot
actionprevents (blocks) untrusted coderecords / logs what loaded
mechanismvalidates digital signatures (UEFI)hashes components into the TPM
on tamperingblocks the component, alertslogs it, does not halt
purposestop boot-level attacksattestation / detection

Secure Boot prevents; Measured Boot detects. Measured Boot needs a TPM to store its measurements; Secure Boot relies on UEFI signatures and works only with Windows.

host-based encryption by OS

OStoolnotes

SED vs host-based: a self-encrypting drive runs encryption on the drive's own engine (keys never leave the hardware, no CPU hit); host-based encryption is managed by the OS.

anonymization techniques

techniquehowreversible?
Data maskingplaceholder values, same formatno
Tokenizationtokens referencing data stored elsewhereyes (via mapping)
Aggregation & bandinggroup values into rangesno

Anonymization vs sanitization: anonymization keeps data usable while hiding identities; sanitization destroys data (overwriting/SDelete, degaussing for magnetic media only, physical destruction). Deleting or formatting is NOT sanitization.

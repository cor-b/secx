Indication Analysis

Digging into the artifacts an incident leaves behind: infrastructure and metadata, volatile and stored data, reversed code and detonated malware, coding fingerprints, and cloud workloads.

overview

Domain 4 (Security Operations), Objective 4.4: analyze data and artifacts in support of incident response. Indication analysis examines components and behaviors to understand threats, investigating infrastructure, data patterns, storage, malware, and code. The section covers infrastructure analysis, metadata analysis, volatile/non-volatile storage analysis, reverse engineering, malware analysis, code stylometry, and the cloud workload protection platform.

area 01

Examine the System

Infrastructure analysis (JTAG, host, network), metadata analysis (files, media, email), volatile/non-volatile storage analysis.

area 02

Examine the Code

Reverse engineering (byte/binary code, disassembly, decompilation), malware analysis (sandboxing, detonation, IoC), code stylometry.

area 03

Protect the Cloud

Cloud workload protection platform (CWPP): detection/response, integration, multi-cloud.

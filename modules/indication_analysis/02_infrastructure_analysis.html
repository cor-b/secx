Infrastructure Analysis

Examining the three layers of a system for compromise: down at the hardware (JTAG), on the endpoints (host), and across the wire (network).

overview

Infrastructure analysis examines hardware, software, and network components to detect vulnerabilities or compromise. Three concepts: JTAG interfaces (hardware-level debugging/testing of embedded systems), host analysis (scrutinizing endpoints for malware/misconfig), and network analysis (traffic patterns and connections).

concept 01

JTAG Interfaces

Hardware interface for debugging/testing embedded systems. Detect tampered firmware/circuits at the hardware level.

concept 02

Host Analysis

Examine endpoints for IoCs (malware, misconfig, odd behavior). Tools: Sysmon, EnCase, Autopsy, SIEM.

concept 03

Network Analysis

Scrutinize traffic/connections for anomalies. Wireshark (full packets) vs NetFlow (metadata only).

JTAG interfaces

JTAG (Joint Test Action Group) is a hardware interface standard for testing and debugging embedded systems, giving direct, low-level access to a device's internals. It's valuable for detecting hardware-level vulnerabilities, insecure firmware, or tampered circuits, e.g. investigating a network switch whose firmware appears modified without authorization, recovering hardware-level logs, and restoring it to a known secure state. It secures even the hardware layer, reducing the attack surface.

host analysis

Host analysis examines individual endpoints (servers, workstations) to detect indicators of compromise, malware, misconfigurations, unusual behavior. Example: an endpoint running slow with files modified without authorization, a security analyst inspects logs, file changes, and processes (using Sysmon, or forensic tools like EnCase, Autopsy) and finds malware that opened an unauthorized channel for exfiltration, then isolates and cleans it. Antivirus, file integrity monitoring, and SIEM platforms automate detection.

network analysis

Network analysis scrutinizes data traffic, communication patterns, and connections to find malicious activity, revealing data breaches, DoS attacks, or unauthorized connections. Example: unusual off-peak traffic where a server contacts an external IP at 2am, packet inspection reveals malware exfiltrating data. Two key tools differ in depth:

Wireshark
Captures full packet data (header + payload) for detailed content/structure analysis, great for investigating specific incidents, but storage- and time-intensive.
NetFlow
Captures metadata only (source/destination IPs, ports, protocols), not the content. More efficient for monitoring large networks over time, but less detailed.
EXAM TIPS

what to know for the exam

  • Three layers: JTAG (hardware), host analysis (endpoints), network analysis (traffic).
  • JTAG = hardware interface for debugging/testing embedded systems; detects firmware/circuit tampering.
  • Host analysis tools: Sysmon, EnCase, Autopsy, SIEM; finds endpoint IoCs.
  • Wireshark captures full packets (header + payload, detailed but heavy); NetFlow captures metadata only (efficient, less detail).
  • Match the need: deep single-incident analysis → Wireshark; broad long-term monitoring → NetFlow.
decision rules
  • Scenario debugs a tampered embedded device's firmware → JTAG interface
  • Scenario inspects an endpoint's processes/logs for malware → host analysis
  • Scenario needs full packet content for one incident → Wireshark
  • Scenario monitors traffic metadata across a large network → NetFlow

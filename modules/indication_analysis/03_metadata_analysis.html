Metadata Analysis

The data about the data. Examining file timestamps, media EXIF, and email headers to uncover origin, manipulation, and malicious intent.

overview

Metadata analysis examines the underlying data about files, media, or communications to uncover origin, manipulation, or malicious intent. Three concepts: files and filesystems (timestamps, permissions), images/audio/video (device, GPS, editing history), and email header analysis (sender, path, authentication).

concept 01

Files & Filesystems

Creation/modification times, permissions, ownership. Tools: stat (Linux), Windows properties, Autopsy, FTK Imager. Spot tampering/unauthorized access.

concept 02

Images/Audio/Video

Device model, GPS coordinates, editing history. Tool: ExifTool. Verify authenticity, trace source.

concept 03

Email Headers

Received fields, Message-ID, Return-Path, SPF/DKIM/DMARC. Tools: MXToolbox, Gmail/Outlook. Detect spoofing/phishing.

files and filesystems

File metadata includes creation date, modification history, and user permissions, revealing tampered or suspicious files through inconsistencies or unusual access. Tools: stat in Linux (or Windows file properties) show timestamps, permissions, and ownership; forensic tools like Autopsy or FTK Imager extract metadata from disk images to reconstruct a security incident (e.g. tracing how malware was planted or finding hidden files with modified attributes).

images, audio, and video

Media metadata can include device model, GPS coordinates, and editing history. The tool ExifTool displays this (camera model, creation date, GPS, editing software). It's used in digital forensics and authenticity verification, e.g. metadata revealing an "original" photo was actually edited, or a video edited after the alleged event, challenging its authenticity and helping establish chain of custody.

email header analysis

Email headers reveal a message's origin and transmission path. Key fields: Received (the server path), Message-ID, and Return-Path, which can show whether an email was spoofed. Tools like MXToolbox or built-in client header analysis (Gmail's "Show original," Outlook) trace the journey. In a phishing attack, headers can show an email originated from an unexpected region or lacks proper authentication (SPF, DKIM, DMARC). The lesson's demo analyzed a fake Chase email and found four malicious indicators: receipt from blacklisted suspicious IPs, an SPF failure, missing DKIM/failed DMARC, and a misaligned Reply-To (secure-chase.login.com vs chase.com), confirming a phishing attempt.

EXAM TIPS

what to know for the exam

  • Three metadata domains: files/filesystems, images/audio/video, email headers.
  • File metadata (stat, Autopsy, FTK Imager) reveals timestamps/permissions, exposing tampering.
  • Media metadata (ExifTool) reveals device, GPS, and editing history for authenticity/source tracing.
  • Email headers (Received, Message-ID, Return-Path) plus SPF/DKIM/DMARC checks detect spoofing/phishing (MXToolbox).
  • Phishing red flags in headers: blacklisted IPs, SPF failure, missing DKIM/failed DMARC, misaligned Reply-To.
decision rules
  • Scenario checks file timestamps for unauthorized changes → file/filesystem metadata
  • Scenario reads GPS/device data from a photo → image metadata (ExifTool)
  • Scenario traces an email's path and finds SPF/DMARC failures → email header analysis
  • Scenario finds a Reply-To that doesn't match the From domain → spoofing indicator (email header)

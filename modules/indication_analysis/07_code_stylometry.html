Code Stylometry

Code has a fingerprint. Analyzing a developer's coding style to recognize evolving variants, find shared code across samples, and trace malware back to its author.

overview

Code stylometry analyzes a developer's coding style to identify unique patterns for malware attribution or tracing software origin. Three concepts: variant matching (similarities between versions of the same malware family), code similarity (shared structures/functions across samples), and malware attribution (linking malware to a threat actor via coding habits).

concept 01

Variant Matching

Spot similarities between versions of the same malware family despite modifications made to evade detection. Recognize evolving threats fast.

concept 02

Code Similarity

Compare code segments across samples for shared structures/functions, linking seemingly unrelated software (or detecting plagiarism).

concept 03

Malware Attribution

Link malware to a developer/group by unique coding habits, reused libraries, preferred algorithms. Turns stylometry into cyber forensics.

variant matching and code similarity

Variant matching identifies similarities between different versions or variants of the same malware family. Attackers tweak code structure or functionality to evade signature-based detection while keeping the core malicious behavior; variant matching links these back to the family (e.g. a new strain with a different payload but the same coding style), enabling rapid identification and response. Code similarity compares code segments across samples to find shared structures, techniques, or functions, linking seemingly unrelated software, detecting threats by comparison to a known-malware database, and even spotting software plagiarism. Example: a new malware sharing the same encryption algorithm as a known banking trojan links it to a specific group.

malware attribution

Malware attribution links a piece of malware to a specific developer, group, or threat actor by analyzing unique coding habits and patterns, markers like coding style, reused libraries, or preferred algorithms. It turns code stylometry into a cyber forensics tool. Knowing who is behind an attack helps predict future risks, adapt defenses, and collaborate with law enforcement or share intelligence. Example: ransomware sharing a particular encryption library and a distinct error-logging style with past attacks lets the team confidently attribute it to a known group and prepare defenses for that group's TTPs.

EXAM TIPS

what to know for the exam

  • Code stylometry = analyzing coding style/patterns for attribution and origin tracing.
  • Variant matching = similarities across versions of the SAME malware family (catches evolving strains).
  • Code similarity = shared structures/functions ACROSS samples (links unrelated software, detects plagiarism).
  • Malware attribution = linking malware to a specific actor by unique coding habits/reused code.
  • Distinguish variant matching (same family) from code similarity (across different samples).
decision rules
  • Scenario links a new strain to an older one in the same family → variant matching
  • Scenario finds shared code across unrelated samples → code similarity
  • Scenario traces malware to a known threat group by coding habits → malware attribution
  • Scenario detects copied code without attribution → code similarity (plagiarism)

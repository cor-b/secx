Update Management

Keeping every layer patched, from the firmware in the hardware up through hypervisors, operating systems, and apps. The catch: firmware and software updates don't ride along with the OS.

overview

Update management ensures all system components are regularly updated and patched. Five areas: firmware, system images, hypervisors, operating systems, and software. Key theme: firmware and software updates are managed separately from OS updates and are easily overlooked.

layer 01

Firmware & Images

Firmware: embedded hardware software (often manual, separate; Dell Command Update). System images: secure baseline for deployments (SCCM, vSphere).

layer 02

Hypervisors

Patch the virtualization layer to prevent VM escape (vCenter, Hyper-V Manager). An unpatched hypervisor risks the whole virtual environment.

layer 03

OS & Software

OS updates (Windows Update, YUM/APT, SCCM) and software updates (separate from OS/firmware) close known vulnerabilities.

firmware and system images

Firmware
Embedded software controlling hardware-OS communication. Critical for network gear, storage, IoT. Often not automatic, it needs separate management (e.g. Dell Command Update) and is easily overlooked since it's not part of OS updates.
System images
Comprehensive backups of a fully configured OS/environment, providing a consistent baseline for deployments. Regularly patch them so each deployment includes the latest fixes (tools: SCCM, VMware vSphere templates). Outdated images multiply vulnerabilities across the network.

hypervisors

Hypervisors host VMs and manage virtual environments. They need frequent patching to address vulnerabilities, managed centrally with tools like VMware vCenter and Microsoft Hyper-V Manager. An unpatched hypervisor is a major risk: an attacker could perform a VM escape, breaking out of one VM to reach others or the host, severe in multi-tenant environments.

operating systems and software

Operating systems need regular updates for vulnerabilities, stability, and compatibility, via Windows Update, YUM/APT (Linux), Apple Software Update, or centralized SCCM. Unpatched OSes are prime targets for malware/ransomware. Software (applications, utilities) also needs regular updates, but these don't come packaged with OS or firmware updates, so they must be handled separately. Outdated apps (browsers, productivity tools) provide entry points and cause compatibility issues.

EXAM TIPS

what to know for the exam

  • Five update areas: firmware, system images, hypervisors, operating systems, software.
  • Firmware and software updates are SEPARATE from OS updates and easily overlooked, firmware is often not automatic.
  • Unpatched hypervisors risk VM escape (breaking out of one VM to others/the host).
  • Outdated system images spread vulnerabilities across every deployment made from them.
  • SCCM is a recurring centralized tool (patches, images, OS).
decision rules
  • Scenario patches embedded hardware software → firmware update (separate, often manual)
  • Scenario prevents breaking out of a VM to the host → hypervisor patching (VM escape)
  • Scenario deploys workstations from a secure baseline → system image management
  • Scenario patches an app that isn't covered by OS updates → software update (handled separately)

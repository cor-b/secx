Encryption Techniques

Two techniques that go beyond plain confidentiality: AEAD also guarantees integrity and authenticity, and envelope encryption protects the keys with more keys.

overview

Two advanced techniques: AEAD (Authenticated Encryption with Associated Data) provides confidentiality, integrity, and authenticity at once, for both the encrypted message and unencrypted-but-protected associated data. Envelope encryption uses two layers: a data key encrypts the data, then a master key encrypts the data key.

technique 01

AEAD

Confidentiality + integrity + authenticity in one. Protects the encrypted message and the visible-but-protected associated data. Uses AES-GCM, a nonce, and an auth tag.

technique 02

Envelope Encryption

Data key encrypts the data; a master key encrypts the data key. Two layers, like a locked box inside a safe.

authenticated encryption with associated data (AEAD)

AEAD keeps data private and proves it wasn't tampered with and came from a legitimate source. It protects both the encrypted data and the associated data (additional authenticated data), information that stays unencrypted but must not be altered. Think of a sealed package (encrypted data) with a visible-but-protected label (associated data): tampering with either breaks the security seal (the authentication tag) and alerts the receiver.

AEAD is realized with modes like AES-GCM (Galois/Counter Mode). It takes four inputs: a secret key, a nonce/IV (a unique, single-use value ensuring identical plaintext produces different ciphertext), the plaintext, and optional associated data. It outputs ciphertext plus an authentication tag the receiver uses to verify integrity and authenticity.

envelope encryption

Envelope encryption uses two layers. A data key encrypts the data; that data key is then encrypted by a stronger master key (often managed separately with asymmetric crypto like RSA). The analogy: lock valuables in a box (data key), then put the box in a safe (master key). It's common in cloud services and secure apps because the data key can be rotated frequently without touching the master key, making key management flexible while keeping both data and keys protected.

EXAM TIPS

what to know for the exam

  • AEAD = confidentiality + integrity + authenticity together, including for the unencrypted associated data. AES-GCM is the classic AEAD mode.
  • AEAD inputs: key, nonce/IV (unique per use), plaintext, optional associated data → outputs ciphertext + authentication tag.
  • Envelope encryption = data key encrypts data; master key encrypts the data key (two layers).
  • Envelope encryption lets you rotate the data key often without changing the master key.
decision rules
  • Scenario needs encryption that also detects tampering and proves authenticity → AEAD
  • Scenario protects a visible header/label alongside encrypted content → AEAD associated data
  • Scenario encrypts a data key with a master key → envelope encryption
  • Scenario names AES-GCM → AEAD mode

Collaborative Considerations

Two ways multiple parties can compute on private data without exposing it: process data while it stays encrypted, or jointly compute a result without sharing inputs.

overview

Collaborative cryptography lets parties compute and share without giving up privacy. Homomorphic encryption performs computations on encrypted data without decrypting it. Secure multi-party computation (SMPC) lets several parties jointly compute a result from their private inputs without revealing those inputs to each other.

technique 01

Homomorphic Encryption

Compute on encrypted data without decrypting it; only the data owner's private key can reveal results. Great for outsourcing to the cloud.

technique 02

Secure Multi-Party Computation

Multiple parties jointly compute a result from private inputs; no one learns anyone else's input, only the final outcome.

homomorphic encryption

Homomorphic encryption allows computation on data without decrypting it first, so an organization can outsource processing to a cloud provider that operates on the encrypted data but can never read it (the private key is never shared). Three types by capability:

Partially homomorphic
Supports one operation (e.g. addition or multiplication) on the encrypted data.
Somewhat homomorphic
Supports a limited set of operations a limited number of times.
Fully homomorphic
Supports any computation on the encrypted data while keeping it secure.

It's invaluable for finance and healthcare, but currently very slow; Microsoft and IBM are working to make it practical. Exam takeaway: homomorphic encryption processes encrypted data without decrypting it, and only the owner's private key can decrypt the results.

secure multi-party computation (SMPC)

SMPC lets multiple parties jointly compute a function over their private inputs without revealing those inputs to each other, protecting data even from the other participants. Example: several companies compute the average of their sales without exposing individual figures, each inputs its data, everyone sees only the average. It uses techniques like Secure Function Evaluation (jointly evaluate a known function without revealing inputs) and Private Function Evaluation (compute without revealing inputs or the function). It's ideal for joint data analysis, secure voting, and any case where data must stay confidential while still contributing to a shared result.

The difference: homomorphic encryption keeps data encrypted during computation; SMPC keeps each party's plaintext input private from the others.

EXAM TIPS

what to know for the exam

  • Homomorphic encryption = compute on encrypted data without decrypting it; only the owner's private key reveals results.
  • Homomorphic types: partial (one operation), somewhat (limited ops), fully (any computation).
  • SMPC = multiple parties jointly compute a result from private inputs; no one sees the others' inputs, just the outcome.
  • Key distinction: homomorphic keeps DATA encrypted during processing; SMPC keeps each party's INPUT private from the others.
  • Homomorphic encryption's main drawback today is being very slow.
decision rules
  • Scenario has a cloud provider compute on data it can never read → homomorphic encryption
  • Scenario has companies compute a shared average without revealing their numbers → secure multi-party computation
  • Scenario needs any operation on data that stays encrypted throughout → fully homomorphic encryption
  • Scenario protects inputs from the other participants, not just outsiders → SMPC

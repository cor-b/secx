Network Issues

Three ways traffic goes wrong: filter rules that block the right things or allow the wrong ones, resources running dry, and attackers deliberately running them dry.

overview

Network issues disrupt the normal flow of data, affecting connectivity, performance, or security. Three concepts: network access control list (NACL) issues (misconfigured rules block legitimate or allow harmful traffic), resource exhaustion (depleted bandwidth/memory/CPU), and DDoS attacks (flooding from many sources to cause resource exhaustion and outage).

issue 01

NACL Issues

Misconfigured rules: too strict blocks legit traffic, too permissive allows attacks. Audit and fine-tune; monitor (SolarWinds, CloudWatch).

issue 02

Resource Exhaustion

Bandwidth/memory/CPU depleted by demand surges or poor management (no load balancing) → slowdowns, outages.

issue 03

DDoS

Flood from many compromised sources to exhaust resources and deny service. Mitigate with DDoS services, firewalls, distribution.

network access control list (NACL) issues

A NACL is a set of rules permitting or denying traffic by source/destination IP and protocol. Misconfiguration cuts both ways. Too strict: a NACL might accidentally block legitimate traffic (filtering out a remote employee's IP range), cutting access to internal apps, often unnoticed until many users are affected. Too permissive: failing to deny untrusted/known-bad IPs leaves open ports for attackers, exposing data to malware and unauthorized access. Mitigate with regular auditing and fine-tuning, balancing security and accessibility, and automated monitoring tools (SolarWinds, Cisco Prime, AWS CloudWatch) to catch errors early.

resource exhaustion

Resource exhaustion is the depletion of critical resources, bandwidth, memory, or processing power, causing performance issues or outages. Causes include excessive demand (a usage surge at peak hours consuming all bandwidth) and inefficient resource management (poorly optimized apps, outdated hardware, unbalanced workloads). For example, a server without proper load balancing receives more traffic than it can handle and slows, lags, or crashes. Mitigation: proactive monitoring, scaling resources to demand (more bandwidth, hardware upgrades, app optimization), and using load balancers and traffic prioritization to keep critical services running.

distributed denial of service (DDoS)

DDoS is the most common deliberate form of resource exhaustion: attackers flood a target with traffic from multiple sources (thousands of compromised devices), overwhelming capacity so legitimate users can't access services, e.g. millions of fake requests crashing an e-commerce site during sales. Consequences include lost revenue, reputation damage, and downtime; attackers often target networks with weak defenses (poorly configured NACLs, outdated firewalls), and the flood can overwhelm routers and switches. Mitigation: dedicated DDoS mitigation services that detect and filter malicious traffic, firewalls/IDS plus properly configured NACLs, and distributing resources across multiple data centers to avoid a single point of failure.

EXAM TIPS

what to know for the exam

  • NACL issues: too strict blocks legitimate traffic; too permissive allows attacks. Audit and monitor regularly.
  • Resource exhaustion = bandwidth/memory/CPU depleted; from demand surges or poor management (missing load balancing).
  • DDoS = flood from many sources to cause resource exhaustion and deny service.
  • DDoS mitigation: dedicated scrubbing services, firewalls/IDS, well-configured NACLs, and distributing across data centers.
  • A weak NACL can fail to block DDoS traffic, compounding the outage.
decision rules
  • Scenario blocks legitimate users due to an over-strict filter → NACL misconfiguration
  • Scenario crashes a server from an un-balanced traffic load → resource exhaustion
  • Scenario floods a site from thousands of devices → DDoS
  • Scenario distributes resources across data centers to survive a flood → DDoS mitigation

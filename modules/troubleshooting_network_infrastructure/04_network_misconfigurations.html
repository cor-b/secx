Network Misconfigurations

Two slow-burning problems: routing left unsecured (open to hijacking), and configurations that quietly drift away from their intended state over time.

overview

Network misconfigurations are incorrect settings on network devices that cause performance issues, security holes, or connectivity failures. Two concepts: insecure routing (routing protocols lacking authentication/encryption, open to hijacking and spoofing) and configuration drift (gradual, often undocumented changes that create inconsistencies and vulnerabilities).

issue 01

Insecure Routing

No authentication/encryption between routers → route hijacking, spoofing, route injection. Worst with weak protocols like RIP.

issue 02

Configuration Drift

Gradual undocumented changes accumulate → inconsistent policies, instability, security gaps. Fix with automation and audits.

insecure routing

Insecure routing occurs when routing protocols or configurations lack proper security. The key weakness is no authentication between routers, letting attackers inject false routing information. This enables route hijacking (redirecting traffic to malicious servers) and route spoofing (fake routes disrupting traffic). For example, an unsecured BGP router can be tricked into accepting and propagating malicious routes, sending legitimate traffic through the attacker's infrastructure for interception.

Outdated/weak protocols make this worse: RIP lacks encryption and authentication and is easily manipulated. Remediation:

  • Replace weak protocols (RIP) with secure ones that support authentication, like OSPF.
  • Enable authentication in OSPF/BGP so routers only accept routes from trusted sources.
  • Use encryption (IPsec) to protect routing updates over public/untrusted networks.
  • Regularly review router configs and disable outdated protocols.

configuration drift

Configuration drift is the gradual change of device configurations over time, from updates, manual fixes, or environmental factors, creating inconsistencies. A common source is undocumented manual changes: small adjustments accumulate (an ACL changed on one router but not synced to others), producing inconsistent security policies. Drift also harms stability: a drifted load-balancing config can overburden some servers and idle others, risking outages.

To combat drift: use automated configuration management tools (Cisco Network Services Orchestrator, Ansible) that flag and correct unauthorized changes, keep regular config backups, and run periodic audits to catch drift early. The transcript's example: an insecure RIP protocol left active due to drift, exposing the network to route injection/spoofing; remediation is to deactivate it, switch to authenticated OSPF, and add automated config management.

EXAM TIPS

what to know for the exam

  • Insecure routing = routing without authentication/encryption → route hijacking, spoofing, injection.
  • Replace weak protocols (RIP) with OSPF + authentication; protect updates with IPsec.
  • BGP without authentication is a classic insecure-routing target.
  • Configuration drift = gradual, often undocumented config changes causing inconsistency and security gaps.
  • Fix drift with automated config management (Ansible, Cisco NSO), backups, and periodic audits.
decision rules
  • Scenario injects false routes into an unauthenticated BGP/RIP setup → insecure routing
  • Scenario finds settings slowly diverged from baseline over months → configuration drift
  • Scenario replaces RIP with authenticated OSPF → insecure routing remediation
  • Scenario deploys Ansible to flag unauthorized config changes → drift remediation

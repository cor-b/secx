Data Protection

Five techniques for safeguarding data: swapping it for tokens, erasing it by destroying keys, hiding its meaning, encrypting it, and packaging it for safe transport.

overview

Data protection safeguards data from unauthorized access, alteration, or loss. Five techniques recur: tokenization replaces sensitive data with non-reversible tokens, cryptographic erase deletes data by destroying its encryption keys, obfuscation hides meaning without cryptography, cryptographic obfuscation uses encryption to make data unreadable, and serialization converts data into a portable format that can be accurately reconstructed.

tokenization

Tokenization replaces sensitive data with unique non-sensitive identifiers called tokens. The token is a placeholder that cannot reveal the original data without access to the secure system that holds the data-to-token mapping, making it effectively non-reversible outside that environment. Even if a token is intercepted, it is meaningless.

NFC payment systems (Apple Pay, Google Pay, Samsung Pay) use tokenization: the wallet stores a token representing the card, not the card number, and transmits the token at the terminal. Only the secure tokenization service can map it back to the real card for authorization. Tools like Thales CipherTrust and TokenEx manage tokenization.

cryptographic erase

Cryptographic erase securely deletes data by encrypting it and then destroying the encryption keys, rendering the data irrecoverable without overwriting the whole drive. The drive itself stays functional and can be re-encrypted with new keys and reused, a major advantage for decommissioning and repurposing hardware.

It is especially preferred for SSDs, where traditional overwriting is less effective because of how data is spread across memory cells. Modern enterprise HDDs and SSDs, particularly self-encrypting drives, include built-in cryptographic erase (Seagate, Western Digital, Samsung), and third-party Secure Erase tools cover drives without native support.

obfuscation vs cryptographic obfuscation

Obfuscation
Hides the meaning of data using encoding or scrambling, not cryptographic algorithms. A common form is masking, such as showing only the last four digits of a Social Security Number. Masking is generally non-reversible, it hides rather than transforms. Used where full encryption is unnecessary.
Cryptographic obfuscation
Goes further, using encryption algorithms to transform data into an unreadable format. Backed by cryptographic security, it is much harder to reverse. Often used to protect software code from reverse engineering. Tools include VeraCrypt and obfuscating compilers.

serialization

Serialization converts data (objects, arrays, data models) into a structured format that can be stored or transmitted and later reconstructed accurately. Common formats are JSON and XML (human-readable, used for web services) and binary (fast and efficient). Deserialization reverses the process. Examples: Python's pickle, Java's Serializable interface, and .NET's JsonSerializer. Serialization preserves data integrity across systems and platforms by keeping all structure and values intact.

An e-commerce flow uses several at once: tokenization replaces card data, obfuscation conceals it during use, serialization stores it so it can be reconstructed, and cryptographic erase destroys it when no longer needed.

EXAM TIPS

what to know for the exam

  • Tokenization swaps sensitive data for tokens; reversible only through the secure mapping system. Think NFC/mobile payments.
  • Cryptographic erase = encrypt then destroy the keys. Fast, irrecoverable, drive stays reusable. Preferred for SSDs.
  • Obfuscation hides meaning WITHOUT cryptography (e.g. masking last-4 of an SSN). Cryptographic obfuscation USES encryption (e.g. protecting code).
  • Serialization converts data to JSON/XML/binary for storage or transit; deserialization rebuilds it.
  • Masking (a form of obfuscation) is generally non-reversible; tokenization is reversible only via the mapping.
decision rules
  • Scenario involves replacing card numbers with placeholders mapped elsewhere → tokenization
  • Scenario involves securely wiping an SSD quickly while keeping the drive usable → cryptographic erase
  • Scenario involves showing only the last four digits of a number → obfuscation (masking)
  • Scenario involves encrypting source code to prevent reverse engineering → cryptographic obfuscation
  • Scenario involves converting an object to JSON to send over a network → serialization

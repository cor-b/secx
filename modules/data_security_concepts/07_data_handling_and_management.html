Data Handling and Management

Two ends of the lifecycle: making data safe to analyze by stripping identities, and making it safe to discard by destroying it for good.

overview

Data handling and management protects data across its lifecycle. Data anonymization (de-identification) modifies data to prevent identifying individuals while keeping it usable for analysis. Data sanitization securely destroys data so sensitive information cannot be recovered before storage media is disposed of, repurposed, or transferred.

concept 01

Data Anonymization

Strips identifying details so individuals cannot be traced, while keeping data analyzable. Techniques: masking, tokenization, aggregation and banding.

concept 02

Data Sanitization

Securely destroys data before media is disposed of or reused. Methods: overwriting, degaussing, physical destruction.

data anonymization

Anonymization, also called de-identification, removes the ability to uniquely identify individuals while letting the data still be analyzed. To report how many students pass an exam you need the totals, not the names. Common techniques:

Data masking
Replaces real data with generic or placeholder values while preserving format. Substituting real card numbers with placeholder digits keeps the structure but hides the value. Masking is not reversible.
Tokenization
Replaces sensitive data with tokens that reference the original stored elsewhere. Unlike masking, tokenization is reversible, allowing re-identification when a legitimate business need arises.
Aggregation and banding
Groups data into ranges to generalize individual values. Reporting that 20 employees earn $40,000–$50,000 hides each exact salary while still giving useful insight.

Anonymization is not foolproof: small datasets can be re-identified. If a survey's demographic bands match only one person in a small company, "anonymous" responses can be traced back to that individual. Aggregation only protects privacy when the groups are large enough.

data sanitization

Sanitization ensures data is completely removed from media before disposal, repurposing, or transfer. Deleting files or formatting a drive is not enough, deleted files are just marked as free space and formatting only removes file-system references, so recovery tools can still retrieve the data. Real sanitization methods:

Overwriting
Writes random patterns, zeros, or ones over the data three, seven, or more times. Replaces the original data with meaningless information. Microsoft's Sysinternals SDelete overwrites files with random data.
Degaussing
Uses strong magnetic fields to scramble data on magnetic media. Works only on magnetic media (HDDs, tapes), not SSDs, and permanently destroys the media so it cannot be reused.
Physical destruction
Shreds, crushes, or incinerates storage devices. Makes recovery and reuse impossible. Banks often industrially shred old drives holding financial data.

Sanitization matters for individuals too: before selling a phone, do a factory reset and run a sanitization tool so no personal data remains.

EXAM TIPS

what to know for the exam

  • Anonymization keeps data usable while removing identifiers. Sanitization destroys the data. Don't mix them up.
  • Masking is not reversible; tokenization is reversible (it maps back to original data). Aggregation/banding groups values into ranges.
  • Anonymization can fail on small datasets through re-identification.
  • Deleting or formatting does NOT sanitize, data remains recoverable. Use overwriting, degaussing, or physical destruction.
  • Degaussing works only on magnetic media (not SSDs) and destroys the media permanently. SDelete overwrites files.
decision rules
  • Scenario involves using customer data for analysis without exposing identities → anonymization
  • Scenario needs reversible de-identification for legitimate business use → tokenization
  • Scenario reports salaries in ranges to hide individuals → aggregation and banding
  • Scenario involves disposing of old drives so data cannot be recovered → sanitization
  • Scenario involves wiping HDDs with magnetic fields → degaussing (not for SSDs)

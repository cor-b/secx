Blockchain

A decentralized ledger where each block is chained to the last by its hash. Change one block and the whole chain breaks, making tampering obvious.

overview

Blockchain is a decentralized, distributed ledger that ensures data integrity and transaction transparency by recording transactions in a series of immutable blocks (unchangeable after creation). An immutable database is any database where data, once written, cannot be altered or deleted. Blockchain uses immutability as a core feature but adds decentralization and cryptography on top.

concept 01

Blockchain

Decentralized, distributed ledger maintained by many nodes. Blocks chain together by hash; tampering is detectable across the network.

Immutable Database

Data, once written, cannot be changed or deleted. Preserves a permanent record. Does not require decentralization on its own.

blockchain

Blockchain works like a digital ledger, but unlike a traditional ledger it is decentralized and distributed: no single person or organization controls it. Instead it is maintained by a network of computers called nodes. Each transaction or piece of data is recorded in a block; when a block is full it is linked to the previous one, forming a chain, hence "blockchain."

Its strengths are transparency and security. Once data is recorded it is effectively impossible to change, giving a tamper-proof record. Every participant holds a copy of the chain, so if someone tampers with the data, the rest of the network sees the discrepancy. This makes blockchain ideal where a transparent, trustworthy record matters: financial transactions, voting systems, and supply chain tracking.

how hash linking makes it immutable

Each block is hashed, producing a digital fingerprint of its data. Crucially, each block also stores the hash of the previous block, linking them into a continuous chain. The first block, with no predecessor, is the genesis block (block 0), which typically contains no transactions.

If any data in a block is altered, even slightly, that block's hash changes, which breaks its link to the next block and alerts the network. To successfully alter one block, an attacker would have to change every subsequent block, recalculate all their hashes, and control the majority of the network's computing power, which is practically impossible. This dependency on previous-block hashes is what makes blockchain immutable.

blockchain vs immutable database

The two are related but not the same. An immutable database simply guarantees that once data is written it cannot be modified, providing a reliable permanent record, valuable for compliance, auditing, and legal contexts. It does not need to be distributed or decentralized. Blockchain is a specific application of an immutable database that also relies on decentralization and cryptography to manage data securely and transparently across a network.

EXAM TIPS

what to know for the exam

  • Blockchain = decentralized, distributed, immutable ledger maintained by nodes; each block linked to the previous by its hash.
  • Immutability comes from hash linking: changing a block changes its hash and breaks the chain, exposing tampering.
  • The genesis block is block 0, the first block, and contains no transactions.
  • An immutable database guarantees write-once data but does NOT require decentralization or cryptography. Blockchain adds both.
  • Good fits: supply chain tracking, financial transactions, voting, any tamper-proof transparent record.
decision rules
  • Scenario needs a tamper-proof, transparent record shared across many parties → blockchain
  • Scenario needs write-once records but no network/decentralization → immutable database
  • Scenario asks what links blocks together → the previous block's hash
  • Scenario asks why altering one block fails → it breaks the hash chain for all later blocks
  • Scenario names the first block with no transactions → genesis block

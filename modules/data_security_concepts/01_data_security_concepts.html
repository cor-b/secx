Data Security Concepts

Applying cryptography across the data lifecycle: proving integrity, protecting data in every state, and meeting privacy obligations.

overview

Domain 3, Objective 3.8: given a scenario, apply appropriate cryptographic use cases and techniques. Protecting data means keeping it accurate and trustworthy, shielding it whether it is stored, moving, or in use, and handling it in line with privacy law. This section covers data integrity, integrity use cases, blockchain, data protection techniques, data state protection, data handling and management, and data compliance and privacy.

theme 01

Integrity

Hashing proves data has not changed; provenance and code signing prove software is trustworthy; blockchain makes records immutable.

theme 02

Protection

Tokenization, cryptographic erase, obfuscation, cryptographic obfuscation, and serialization safeguard data from exposure and loss.

theme 03

States

Data at rest, in transit, and in use each need their own protection: AES, TLS/IPsec, and in-memory encryption.

theme 04

Governance

Anonymization and sanitization manage data across its lifecycle; GDPR and CCPA set the legal and regulatory guardrails.

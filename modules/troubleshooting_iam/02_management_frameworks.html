Management Frameworks

How identities and access controls are managed, monitored, and maintained: proving someone is who they claim, then handing out elevated power only when it's needed.

overview

Management frameworks define how identities and access are managed within an organization. Core processes: authentication (verify identity) and authorization (determine access). Two key concepts: identity proofing (verifying a claimed identity is accurate, usually at account setup or password reset) and privileged identity management (PIM) (granting elevated permissions only when necessary and monitoring privileged accounts).

identity proofing

Identity proofing confirms a person's identity by asking for additional proof, at onboarding or during a password reset. Knowledge-based questions (where you went to high school, first pet) assume only you know the answer, but this is weak: the 2008 Sarah Palin Yahoo email hack succeeded because the attacker answered security questions using publicly available info. A defense is to use an alternative persona, answering with unique values not findable online. Larger orgs require stronger proof: presenting a photo ID or passport at the IT service desk. Higher security, but slower, so it's a trade-off.

Once verified, the identity is shared across systems through identity propagation. If a system handles its own authentication, only the identity is passed; with SSO, both identity and authentication details transfer. Microsoft Active Directory uses Kerberos ticketing for this. Bridging proprietary and third-party systems may need a centralized identity/authentication translation system handling tickets, SSO, or XML tokens. Protocols like CredSSP (Credential Security Support Provider) can transmit full user credentials and must run over SSL/TLS.

privileged identity management (PIM)

PIM manages and controls access to critical systems by granting elevated permissions only when necessary, the just-in-time principle. Instead of an admin holding powerful permissions permanently, they request temporary elevated access for a specific task; once finished, the permissions are removed, shrinking the window for misuse or accidental damage. PIM also closely monitors privileged accounts: every elevated action is logged and tracked, creating an audit trail that helps detect unauthorized changes and hold privileged users accountable.

EXAM TIPS

what to know for the exam

  • Authentication verifies identity; authorization determines what resources you can access.
  • Identity proofing verifies a claimed identity is accurate, done at account setup or password reset.
  • Knowledge-based questions are weak (Sarah Palin hack); use an alternative persona with non-discoverable answers.
  • Identity propagation shares the verified identity across systems; SSO also shares authentication details (AD uses Kerberos).
  • PIM grants elevated permissions just-in-time and monitors/audits privileged accounts.
decision rules
  • Scenario verifies a new hire's claimed identity at onboarding → identity proofing
  • Scenario grants an admin temporary elevated rights for one task → PIM (just-in-time)
  • Scenario answers security questions with fake-but-memorable values → alternative persona
  • Scenario transmits full credentials and must be encrypted → CredSSP over SSL/TLS

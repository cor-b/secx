WiFi Authentication

Securing access to wireless networks: 802.1X is the gatekeeper, EAP is the flexible framework of methods it runs, and SAE is WPA3's modern, forward-secret handshake.

overview

WiFi authentication secures access to wireless networks. Three concepts: IEEE 802.1X (network access control framework using an authentication server like RADIUS), EAP (Extensible Authentication Protocol, a flexible framework of authentication methods between supplicant and network), and SAE (Simultaneous Authentication of Equals, WPA3's secure password-based handshake with forward secrecy).

concept 01

802.1X

Network access control: gatekeeper that authenticates devices via a RADIUS server. Supplicant → RADIUS client (AP) → RADIUS server.

concept 02

EAP

Flexible framework of auth methods within 802.1X. Replaced PAP/CHAP. Variants range from weak (MD5) to strong (TLS).

concept 03

SAE

WPA3 handshake replacing the WPA2 four-way handshake. Password-based, uses Dragonfly + ECC, gives forward secrecy.

IEEE 802.1X

802.1X (from the Institute of Electrical and Electronics Engineers) is a network access control protocol that authenticates devices before granting access, working on both wireless and wired networks. The flow: a device (the supplicant) wants to join; EAP sets up a secure channel to pass credentials; the router/access point (the RADIUS client) forwards them to the RADIUS server (Remote Authentication Dial-In User Service), which is the final authority, validating against its database or a directory like Active Directory. If valid, the server approves and the client grants access.

EAP (Extensible Authentication Protocol)

EAP is not a single method but a framework supporting many authentication methods, developed to replace weaker PAP (sends credentials in plain text, insecure) and CHAP (challenge/response, never sends the password; MS-CHAP is Microsoft's variant with mutual auth). Five common EAP types:

EAP-MD5 (CHAP)
Uses CHAP within EAP, password-based. Weakest, vulnerable to dictionary attacks with weak passwords.
EAP-TLS
Most secure: digital certificates on both supplicant and server, no passwords, mutual authentication.
PEAP
Protected EAP: server certificate forms an encrypted tunnel; user authenticates inside it (e.g. MSCHAPv2, OTP).
EAP-TTLS
Tunneled TLS: like PEAP but more flexible inner methods; server cert + password client auth. Less secure than EAP-TLS.
EAP-FAST
Uses a protected access credential (PAC) instead of a cert; faster but credential distribution is a challenge; least secure of these.

For the highest security, use EAP-TLS with mutual certificate authentication; if certificates are too costly, PEAP or EAP-TTLS are good alternatives.

SAE (Simultaneous Authentication of Equals)

SAE is WPA3's authentication feature, replacing the WPA/WPA2 four-way handshake (which relied on Diffie-Hellman pre-shared key exchange and was vulnerable to interception, cracking, and replay). SAE is a secure password-based method providing forward secrecy (also called perfect forward secrecy): even if long-term keys are compromised, each session's keys stay secure, so past communications can't be decrypted. It uses the Dragonfly password-authenticated key exchange with elliptic curve cryptography to derive a unique session key per session. A new key every session means a compromised key doesn't affect past or future sessions, making WPA3 far stronger than WPA2/WPA/WEP.

EXAM TIPS

what to know for the exam

  • 802.1X = port/network access control using a RADIUS server; roles: supplicant, RADIUS client (AP), RADIUS server.
  • EAP is a framework, not one method. EAP-TLS (certs on both sides) is the most secure; EAP-MD5 the weakest.
  • PEAP and EAP-TTLS build an encrypted tunnel with a server certificate; EAP-FAST uses a PAC.
  • SAE (WPA3) replaces the WPA2 four-way handshake; password-based, uses Dragonfly + ECC, gives forward secrecy.
  • PAP sends plain text (insecure); CHAP/MS-CHAP use challenge/response.
decision rules
  • Scenario needs the most secure EAP with mutual certs → EAP-TLS
  • Scenario authenticates WiFi devices via a RADIUS server → 802.1X
  • Scenario wants WPA3 forward secrecy in the handshake → SAE (Dragonfly)
  • Scenario can't afford client certificates but wants a tunnel → PEAP / EAP-TTLS

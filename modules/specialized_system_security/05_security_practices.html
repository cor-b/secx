Security Practices

Three layered defenses for systems you often can't patch: wall them off (segmentation), shrink their attack surface (hardening), and watch them constantly (monitoring).

overview

Security practices protect specialized systems through three strategies: segmentation (divide the network into isolated sections to limit attack spread), hardening (reduce the attack surface by disabling unneeded services and patching), and monitoring (continuously watch activity to detect incidents in real time). Together they provide layered protection.

practice 01

Segmentation

Divide the network into isolated sections (VLANs + firewalls) so a breach in one area can't spread. Isolate ICS from corporate network.

practice 02

Hardening

Reduce the attack surface: disable unneeded services, apply patches, configure stronger settings. Fewer entry points.

practice 03

Monitoring

Continuously watch activity for anomalies; SIEM + IDS for real-time alerts on unauthorized access.

segmentation

Segmentation divides a larger network into smaller, isolated sections, so if an attacker compromises one area, they can't easily reach the rest. An office might keep employee devices on a separate network from sensitive financial/customer-data systems. A common tool is the VLAN, logically separating devices at layer 2. It's especially valuable for protecting specialized ICS: a factory isolates its operational systems controlling machinery from the broader corporate network (email, etc.), limiting damage if the corporate side is breached. Firewalls (Cisco ASA, Palo Alto Next-Generation Firewalls) combine with segmentation to enforce which segments may communicate and under what conditions.

hardening

Hardening minimizes a system's vulnerabilities by reducing its attack surface, disabling unnecessary services or features (a locally-accessed database shouldn't have remote login enabled) and applying security patches when vendors release them. For a web server, disable unneeded features/services (e.g. FTP if not required) so there are fewer entry points to exploit. Tools like the Microsoft Security Compliance Toolkit or open-source Lynis identify where systems can be hardened.

monitoring

Monitoring keeps a close watch on network and system activity to spot incidents as they happen, detecting unusual behavior like a traffic spike or unauthorized access attempts. SIEM systems collect and analyze data across the organization to surface issues. For a retailer handling credit cards, real-time monitoring of the payment system flags unauthorized access quickly. Tools: Splunk or Graylog for log collection/analysis and real-time alerts, plus IDS like Snort or Suricata for unusual patterns and unauthorized access.

EXAM TIPS

what to know for the exam

  • Segmentation isolates network sections (VLANs + firewalls) to contain attacks; isolate ICS/OT from the corporate network.
  • Hardening reduces the attack surface: disable unnecessary services, apply patches (tools: MS Security Compliance Toolkit, Lynis).
  • Monitoring watches activity in real time for anomalies; SIEM (Splunk, Graylog) + IDS (Snort, Suricata).
  • These three are layered defense, especially important for systems that can't be patched directly.
decision rules
  • Scenario isolates factory machinery from the corporate LAN → segmentation
  • Scenario disables FTP and unused services on a server → hardening
  • Scenario uses a SIEM for real-time alerts on suspicious access → monitoring
  • Scenario can't patch a legacy device, so it walls it off and watches it → segmentation + monitoring

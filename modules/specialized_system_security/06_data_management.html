Data Management

Pulling scattered data into one place (aggregation), then mining it for threats (analytics), through a SIEM pipeline, with database activity monitoring guarding the stored result.

overview

Data management organizes, protects, and analyzes data to keep its integrity, confidentiality, and availability. Two concepts: aggregation (collecting and combining data from many sources into one comprehensive data set) and data analytics (examining aggregated data to find insights and detect threats). A SIEM drives the pipeline; database activity monitoring protects the stored data.

concept 01

Aggregation

Collect + combine data from many sources into one normalized data set, so patterns are visible across the whole picture, not fragments.

concept 02

Data Analytics

Analyze aggregated data (often via SIEM) for threats. Stream processing for near-real-time; indexing and log curation keep it efficient.

aggregation

Aggregation collects and combines data from multiple sources into one comprehensive data set, so it can be analyzed for patterns, trends, or anomalies. Without it, data is fragmented across systems and you can't see the full picture. For an e-commerce business, aggregating website purchases, app activity, social-media marketing data, and customer-service feedback into one set reveals patterns (customers who use the app more buy higher-value products after specific ads). Technically, aggregation needs a system that collects data in real time or at intervals, normalizes the differing formats, and combines it into a central repository.

data analytics

Data analytics processes aggregated data to detect security incidents and threats, using tools that sift large volumes from endpoints, IAM, network traffic, threat feeds, and cloud platforms. A common platform is the SIEM, which:

  • Normalizes data into a consistent format (different systems use different formats) so it can be searched, indexed, and analyzed centrally.
  • Runs data through a processing pipeline: older batch processing (collected over fixed periods, introduces delay) vs modern stream processing (near real-time, faster detection/response).
  • Indexes data in the central repository to speed searches.
  • Applies log curation to prioritize and filter the most relevant data, collecting everything indiscriminately overwhelms the system and wastes resources.

database activity monitoring (DAM)

Once data is stored, database activity monitoring tools secure it by detecting unauthorized access and suspicious activity and supporting compliance audits. Three methods:

Interception-based
Monitors client-server communication. May miss some data.
Memory-based
A sensor on the database captures SQL statements as executed. Captures more, but can slow the system.
Log-based
Monitors transactions via database server logs. May miss some data.

DAM also supports data classification, DLP, and integrity, combining network sniffing, memory scraping, system tables, and log analysis for a full picture of who's accessing the database.

EXAM TIPS

what to know for the exam

  • Aggregation = combine data from many sources into one normalized data set; you can't see patterns in fragments.
  • Data analytics = analyze the aggregated data for threats, usually via a SIEM.
  • SIEM pipeline: normalize → process (batch vs stream) → index → curate. Stream processing = near real-time.
  • Log curation filters to the most relevant data; collecting everything overwhelms the system.
  • DAM methods: interception-based, memory-based (most data, can slow system), log-based.
decision rules
  • Scenario combines many data sources into one set → aggregation
  • Scenario needs near-real-time threat detection from logs → stream processing (SIEM)
  • Scenario captures SQL as executed but may slow the DB → memory-based DAM
  • Scenario filters to only the most relevant logs → log curation

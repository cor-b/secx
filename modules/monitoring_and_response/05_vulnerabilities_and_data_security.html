Vulnerabilities and Data Security

Two complementary tools: scanners that hunt for weaknesses before attackers do, and data loss prevention that stops sensitive information from leaving.

overview

Vulnerabilities and data security involve finding system weaknesses and protecting sensitive data. Two concepts: vulnerability scans (automated detection of flaws, misconfigs, unpatched software) and data loss prevention (DLP) (monitor and prevent unauthorized transmission of sensitive data).

tool 01

Vulnerability Scans

Automated detection of flaws/misconfigs/unpatched software. Scheduled regularly + after changes. Tools: OpenVAS, Nessus, Qualys.

tool 02

Data Loss Prevention

Monitor data at rest/in transit/in use. Actions: alert, block, quarantine, tombstone. Tools: Symantec DLP, Microsoft Purview.

vulnerability scans

Vulnerability scans identify potential security flaws, unpatched software, misconfigurations, exploitable weaknesses, before attackers do. They should be scheduled and automated (often weekly or daily) and run after significant changes (upgrades, config changes) to catch newly introduced weaknesses; automation reduces human error. Review scan reports to prioritize fixes by severity. Common tools:

  • OpenVAS: open-source network vulnerability scanner.
  • Nessus: commercial, large vulnerability database, customizable plugins (e.g. checking PCI DSS compliance).
  • Qualys: cloud-based, continuous scanning integrated with asset management.

data loss prevention (DLP)

DLP prevents sensitive data (customer records, IP, financials) from being shared with unauthorized parties, monitoring data at rest, in transit, and in use across channels like email, cloud storage, and internal apps. It enforces policies with several possible actions:

Alert
Notify the security team of a violation.
Block
Stop the transmission entirely (e.g. an email with credit card numbers never leaves).
Quarantine
Hold the data in a secure queue for review before release.
Tombstoning
Replace sensitive content with a placeholder (e.g. "redacted") plus information on the policy violation and remediation, so workflows continue without exposing data.

Tools: Symantec DLP (multi-channel: endpoints, email, cloud) and Microsoft Purview (integrated into SharePoint, Teams, Outlook).

EXAM TIPS

what to know for the exam

  • Vulnerability scans find flaws/misconfigs/unpatched software; schedule them regularly and after changes.
  • Scan tools: OpenVAS (open-source), Nessus (commercial, plugins), Qualys (cloud, continuous).
  • DLP monitors data at rest, in transit, and in use; actions: alert, block, quarantine, tombstone.
  • Tombstoning replaces sensitive content with a placeholder + remediation info so workflows continue.
  • DLP tools: Symantec DLP, Microsoft Purview (integrated into Microsoft apps).
decision rules
  • Scenario automatically scans for unpatched software weekly → vulnerability scan
  • Scenario blocks an email containing credit card numbers → DLP (block)
  • Scenario replaces sensitive data with "redacted" and a policy notice → tombstoning
  • Scenario holds a flagged email for review before release → DLP (quarantine)

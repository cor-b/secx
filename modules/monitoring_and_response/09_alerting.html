Alerting

Notifying the team when something's wrong, and the ways alerting goes wrong: false alarms, missed threats, and alerts that never fire at all.

overview

Alerting notifies security teams about potential threats based on predefined rules and triggers. Concepts: alerts associated with vulnerabilities (detected weaknesses, true positives), false positives (benign flagged as threat), false negatives (real threat undetected), malware (infection alerts), and alert failures (missed/delayed notifications).

concept 01

Vulnerability Alerts

Fire when weaknesses are detected (true positives). FIM and SIEMs (Splunk, QRadar) help. Patch before exploitation.

concept 02

False Positives/Negatives

FP = benign flagged (alert fatigue); FN = real threat missed (more dangerous). Tune and update detection.

concept 03

Malware & Alert Failures

Malware alerts (signature/heuristic/behavioral). Failures from misconfig, poor prioritization, overload.

vulnerability alerts and false positives/negatives

Vulnerability alerts fire when a potential weakness is detected (a detected actual vulnerability is a true positive), letting teams patch before exploitation. File integrity monitoring (FIM) alerts on unexpected changes to critical files; SIEMs (Splunk, QRadar) monitor for abnormal behavior. False positives wrongly flag normal behavior, causing alert fatigue, reduced by fine-tuning FIM databases and DLP filters. False negatives are more dangerous, a real threat goes undetected (e.g. an IDS missing a signatureless attack), reduced with continuous IDS/IPS updates and anomaly detection.

malware and alert failures

Malware alerts notify teams of malicious software using signature-based detection (matching known signatures), heuristic analysis (suspicious code patterns), and behavioral monitoring (unusual activity), plus real-time scanning and sandboxing (e.g. quarantining a system found encrypting files via ransomware).

Alert failures happen when an alert is delayed, not generated, or missed, from three causes: misconfiguration (a firewall allowing blocked traffic, outdated SIEM rules), poor prioritization (critical alerts buried among routine ones), and overloaded tools (a SIEM processing millions of events/sec, delaying or crashing). Mitigate with proper configuration, updated rules, robust prioritization, scalable/cloud SIEMs, event filtering, and IPS that takes automatic action when alerts are missed.

EXAM TIPS

what to know for the exam

  • Detected actual vulnerability = true positive; false positive = benign flagged; false negative = real threat missed (most dangerous).
  • FIM alerts on unexpected changes to critical files.
  • Malware detection: signature-based, heuristic analysis, behavioral monitoring (+ sandboxing).
  • Alert failure causes: misconfiguration, poor prioritization, overloaded tools.
  • Mitigate failures with updated rules, prioritization, scalable SIEMs, and IPS auto-action.
decision rules
  • Scenario alerts on an unexpected change to a critical file → FIM vulnerability alert
  • Scenario floods analysts with benign alerts → false positives (alert fatigue)
  • Scenario misses a signatureless attack → false negative
  • Scenario's SIEM crashes under event volume and misses alerts → alert failure (overload)

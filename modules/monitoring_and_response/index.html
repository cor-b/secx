Monitoring and Response

Analyzing data to detect and respond: aggregate analysis, threat intel, log sources, vulnerability/data security, behavior baselines, SIEM event and data management, alerting, prioritization, and reporting.

overview

Domain 4 (Security Operations) OBJ 4.1: analyze data to enable monitoring and response activities. Topics span aggregate data analysis (reduction, correlation, prioritization, trends), threat intelligence sources (feeds, CVE, bug bounty, reports), system log sources (infrastructure, endpoint, application, CSPM), vulnerabilities and data security (scans, DLP), behavior baselines (network/system/user/application), SIEM event and data management, alerting, alert prioritization factors, and reporting and metrics (visualization, dashboards).

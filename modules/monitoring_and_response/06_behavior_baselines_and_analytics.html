Behavior Baselines and Analytics

You can't spot the abnormal until you know the normal. Baselines for the network, the system, the user, and the application define "normal" so anomalies stand out.

overview

Behavior baselines establish normal patterns of activity so anomalies indicating threats can be recognized. Four baseline types: network (traffic patterns), system (resource usage), user (login/access habits), and application/service (usage and performance metrics).

baseline 01

Network

Normal traffic patterns/connections. Snort (IDS), Suricata (IDS/IPS). Spot data exfiltration, C2, DDoS.

baseline 02

System

Normal CPU/memory/disk usage. Windows PerfMon to build; Datadog/Prometheus to monitor. Spot malware, memory leaks.

baseline 03

User (UBA)

Normal login times, file access, devices. Splunk UBA. Spot compromised credentials, insider threats.

baseline 04

Application/Service

Normal response times, users, error rates. AppDynamics. Spot performance issues, DoS, exploited services.

the four baselines

Network baseline
Typical traffic patterns, connections, and peak/low times. Monitored against real-time traffic by Snort (IDS) and Suricata (IDS/IPS). A spike to an unknown external IP may signal data exfiltration or C2; a traffic surge may signal DDoS.
System baseline
Normal CPU, memory, and disk I/O usage. Built with tools like Windows PerfMon, monitored with Datadog/Prometheus. A CPU spike to 100% may mean malware; sustained high memory may mean a memory leak.
User behavior analytics (UBA)
Normal login times, file/system access, and devices. Tracked with Splunk User Behavior Analytics. A late-night login from a foreign IP or sudden access to sensitive files may signal compromised credentials or an insider threat.
Application/service baseline
Normal response times, active users, error rates, resource use. Monitored with AppDynamics. A slowdown or error-rate spike may mean performance issues or a DoS; excessive resource use may mean an exploited service.
EXAM TIPS

what to know for the exam

  • Establish "normal" first, anomalies are deviations from the baseline.
  • Four baselines: network (traffic), system (CPU/memory/disk), user (UBA: logins/access), application/service (response/errors).
  • Network: Snort (IDS) vs Suricata (IDS/IPS). System: PerfMon (build), Datadog/Prometheus (monitor).
  • UBA (Splunk UBA) detects compromised credentials and insider threats via odd login times/locations.
  • A CPU spike to 100% suggests malware; sustained high memory suggests a leak.
decision rules
  • Scenario sees a traffic spike to an unknown external IP → network baseline anomaly
  • Scenario sees CPU jump to 100% with no reason → system baseline anomaly (malware)
  • Scenario sees a user log in at 3am from abroad → user behavior anomaly (UBA)
  • Scenario sees an app's error rate spike → application/service baseline anomaly

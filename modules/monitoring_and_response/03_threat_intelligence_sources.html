Threat Intelligence Sources

Where insight into emerging threats comes from: real-time feeds of indicators, the CVE catalog of known vulnerabilities, bug bounty disclosures, and third-party reports and logs.

overview

Threat intelligence sources provide insight into emerging threats, vulnerabilities, and attacker tactics. Four sources: threat intelligence feeds (real-time IOC data), CVE details (standardized vulnerability identifiers), bug bounty programs (researcher-reported vulnerabilities), and third-party reports and logs (context and analysis).

source 01

Threat Intelligence Feeds

Real-time IOCs (malicious IPs, URLs, hashes). Public (AlienVault OTX) or private (ISACs). Ingested via STIX/TAXII.

source 02

CVE Details

Standardized IDs for known vulnerabilities (CVE-YYYY-NNNN), maintained by MITRE, with CVSS severity scores. Prioritize patching.

source 03

Bug Bounty Programs

Reward researchers for responsible disclosure (HackerOne, Bugcrowd). Apple offered up to $1M for zero-click RCE.

source 04

Third-party Reports/Logs

Context and analysis (Verizon DBIR), plus MSSP/cloud logs (AWS access logs) for continuous monitoring.

feeds and CVE details

Threat intelligence feeds are real-time streams of indicators of compromise (IOCs), malicious IPs, URLs, file hashes, domains. Public feeds (e.g. AlienVault OTX) are free; private feeds (paid, industry-specific, like financial or healthcare ISACs) are tailored. Feeds are ingested into firewalls, IDS, and SIEMs, often via the STIX (Structured Threat Information eXpression) format over the TAXII protocol, to automatically block or alert on malicious indicators.

CVEs are standardized identifiers for publicly known vulnerabilities, formatted CVE-[year]-[number] (e.g. CVE-2021-34527, PrintNightmare), maintained by MITRE and cataloged in the National Vulnerability Database. Each carries a CVSS severity score (0.0–10.0) to prioritize patching.

bug bounties and third-party sources

Bug bounty programs reward independent researchers for responsible disclosure of vulnerabilities to vendors (platforms: HackerOne, Bugcrowd), rewards range from hundreds to over $100,000; Apple offered up to $1 million for a zero-click remote-code-execution iOS flaw. They give vendors access to a global pool of security talent. Third-party reports and logs provide context and analysis: annual reports like the Verizon Data Breach Investigations Report (DBIR) aggregate thousands of incidents, and third-party logs (MSSPs, cloud providers like AWS access logs) can be integrated into a SIEM for continuous monitoring.

EXAM TIPS

what to know for the exam

  • Four sources: threat intelligence feeds, CVE details, bug bounty programs, third-party reports/logs.
  • Feeds deliver real-time IOCs; public (AlienVault OTX) vs private (ISACs); ingested via STIX (format) over TAXII (protocol).
  • CVE = standardized vulnerability ID (CVE-YYYY-NNNN, MITRE); CVSS gives the 0.0-10.0 severity score.
  • Bug bounties reward responsible disclosure (HackerOne, Bugcrowd).
  • Third-party reports (Verizon DBIR) and logs (AWS) add context; can feed a SIEM.
decision rules
  • Scenario auto-ingests malicious IPs/hashes in real time → threat intelligence feed (STIX/TAXII)
  • Scenario references a vulnerability by CVE-YYYY-NNNN → CVE detail
  • Scenario pays a researcher for a reported flaw → bug bounty program
  • Scenario cites the Verizon DBIR for trends → third-party report

Cron and Scheduled Tasks

Making automation run on its own clock: Cron on Linux and Task Scheduler on Windows execute scripts at predefined times so critical security tasks never depend on someone remembering.

overview

Cron and Scheduled Tasks automate execution of scripts at predefined times. Cron (from Chronos, Greek for time) is the Linux time-based job scheduler, configured in the crontab. Scheduled Tasks is the Windows equivalent, managed via Task Scheduler. Both ensure critical processes (scans, backups, patches) run consistently without manual intervention.

tool 01

Cron (Linux)

Time-based job scheduler configured in the crontab. Five time fields: minute, hour, day-of-month, month, day-of-week. Pairs with Bash.

tool 02

Scheduled Tasks (Windows)

Managed via Windows Task Scheduler; daily/weekly/monthly runs. Pairs with PowerShell scripts.

cron

Cron is a time-based job scheduler in Linux, useful for automating repetitive tasks like security scans or log updates. It's controlled by the crontab file, where each line defines a task with five time fields:

  1. Minute
  2. Hour
  3. Day of the month
  4. Month
  5. Day of the week (day 1 = Monday)

An asterisk (*) in a position means "every." Example: 20 20 * * 1-5 /scripts/backup-server runs every weekday (days 1-5) at 8:20 PM (20th hour, 20th minute). Cron integrates with Bash scripts, e.g. comparing current file hashes against known-good hashes to detect unauthorized modifications and triggering an alert on mismatch. Manage it with crontab -e (edit) and crontab -l (list). It reduces human error and keeps updates, backups, and scans running on time.

scheduled tasks

Scheduled Tasks is the Windows time-based scheduler, managed through the Windows Task Scheduler, useful for maintenance scripts, backups, and software updates at set intervals (daily, weekly, monthly). It works alongside PowerShell scripts, e.g. scanning logs for errors, sending alerts, or backing up files, enabling automated workflows so admins don't manually trigger tasks. Like Cron, it reduces workload, human error, and keeps the system secure and maintained.

EXAM TIPS

what to know for the exam

  • Cron = Linux time-based scheduler (from Chronos); Scheduled Tasks = Windows (Task Scheduler).
  • Crontab has five fields in order: minute, hour, day-of-month, month, day-of-week (Monday = 1).
  • An asterisk means "every" for that field; ranges like 1-5 mean weekdays.
  • Cron pairs with Bash; Scheduled Tasks pairs with PowerShell.
  • Both ensure consistent, unattended execution of scans, backups, and patches.
decision rules
  • Scenario schedules a nightly scan on Linux → Cron
  • Scenario automates patch deployment on Windows → Scheduled Tasks
  • Scenario reads "0 7 * * *" → runs daily at 7:00 AM
  • Scenario edits then lists scheduled jobs on Linux → crontab -e / crontab -l

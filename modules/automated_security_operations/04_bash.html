Bash

The Bourne Again Shell: the scripting language for automating Unix and Linux security operations, with its own conventions for storing data and controlling flow.

overview

Bash (Bourne Again Shell) is a Unix shell and command language for writing scripts. It's tailored for Unix/Linux systems (unlike PowerShell for Windows or general-purpose Python), making it ideal for Linux security operations. Its data structures are variables, arrays, and strings; its control structures are conditionals (if/elif/else, case) and loops (for, while).

bash data structures

Variables
Store single values. No keyword needed, assign with = and no spaces around it (username=admin). Everything is treated as a string unless specified. Access the value with $ ($username), printed via echo.
Arrays
Store multiple values, created in quotes separated by spaces. Use zero-based indexing: the first element is position 0. ${servers[0]} retrieves the first element.
Strings
Sequences of characters; can be concatenated and manipulated with special operators (#, %). ${#greeting} gives the length of a string (e.g. "Hello, world!" → 13).

bash control structures

  • if / elif / else: the condition is in square brackets; the block opens with then and closes with fi ("if" backward). Checks like whether user equals admin.
  • for loop: iterates over lists/arrays (${servers[@]} for all elements), with the block marked by do and done.
  • while loop: runs as long as a condition (in square brackets) stays true; block marked with do/done (e.g. ((count++)) increments).
  • case statement: matches a variable against several values, an alternative to many ifs. Each block ends with ;; and the whole block closes with esac ("case" backward).

Combining these, Bash handles complex decision-making and task automation, ideal for managing Unix-based systems (e.g. a script scanning log files using arrays to store results and loops to iterate over entries).

EXAM TIPS

what to know for the exam

  • Bash = Bourne Again Shell, for Unix/Linux automation (PowerShell = Windows, Python = cross-platform).
  • Variables: assign with = and no spaces; access with $; everything is a string by default.
  • Arrays use zero-based indexing; ${arr[0]} is the first element, ${arr[@]} is all.
  • Blocks close with reversed keywords: fi (if), esac (case); loops use do/done.
  • Conditions go in square brackets; ${#string} gives string length.
decision rules
  • Scenario automates tasks on a Linux server → Bash
  • Scenario closes a conditional with "fi" → Bash if statement
  • Scenario matches one variable against many values → case statement
  • Scenario accesses the first array element at position 0 → zero-based indexing

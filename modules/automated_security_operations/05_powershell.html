PowerShell

Windows' scripting language and command-line shell, built around cmdlets and .NET objects that flow through the pipeline, optimized for system administration.

overview

PowerShell is a scripting language and command-line shell for automating tasks and managing configurations in Windows (unlike Unix-based Bash or general-purpose Python). It uses cmdlets (verb-noun commands returning .NET objects) and a pipeline. Data structures: variables, arrays, hash tables, strings, objects. Control structures: if/else, for, foreach, while, switch.

powershell data structures

PowerShell handles data through cmdlets, lightweight verb-noun commands (e.g. Get-EventLog) that perform an action and typically return a .NET object. Objects are a powerful data structure: a log entry object carries properties like TimeWritten and Message that you can access and process directly. The pipe operator (|) sends the output of one command as input to the next, e.g. retrieving the five most recent security-log entries with instance ID 4625 (logon failures), selecting properties, and piping them into Out-File to write logfail.txt. PowerShell also uses arrays for lists (log entries, user accounts) and hash tables for key-value data. Its built-in object handling makes it well-suited to system administration.

powershell control structures

  • Write-Host: prints messages to the screen to show script progress.
  • if / else: decision-making (e.g. check whether the user is an administrator and display a message).
  • for / foreach: repeat tasks over a series; foreach iterates a collection like all log entries, checking each (e.g. for failed logons) and acting, working hand in hand with cmdlets like Get-EventLog.
  • while: repeats while a condition holds.
  • switch: efficiently handles multiple possible values of a variable (e.g. role = admin/user/guest), cleaner than many ifs.

Combining control structures with cmdlets, PowerShell can dynamically retrieve, process, and act on system data, e.g. auditing Active Directory user accounts: arrays store user data, loops iterate, conditionals flag accounts needing attention.

EXAM TIPS

what to know for the exam

  • PowerShell is optimized for Windows administration (Bash = Unix/Linux, Python = cross-platform).
  • Cmdlets are verb-noun commands (Get-EventLog) returning .NET objects with properties.
  • The pipe operator (|) sends one command's output as the next command's input.
  • Data structures: variables, arrays, hash tables, strings, objects. Control: if/else, for, foreach, while, switch.
  • Objects (not plain text) are PowerShell's key data structure, distinguishing it from Bash.
decision rules
  • Scenario audits Active Directory accounts on Windows → PowerShell
  • Scenario uses a verb-noun command returning objects → cmdlet
  • Scenario chains command output into the next command → pipe operator
  • Scenario matches a role variable against admin/user/guest → switch statement

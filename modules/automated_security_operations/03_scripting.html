Scripting

The building blocks shared by every scripting language: a plain-language plan (pseudocode), organized storage (data structures), and the logic that drives decisions and repetition (control structures).

overview

Scripting writes code that automates repetitive tasks for efficiency and consistency. Three core concepts: pseudocode (a plain-language outline of logic before coding), data structures (arrays, lists, dictionaries to organize data), and control structures (loops and conditionals that direct flow). These apply across Bash, PowerShell, and Python.

concept 01

Pseudocode

A plain-language blueprint of a script's logic and steps, no syntax. Like a rough house sketch. Plans the flow, eases team communication.

concept 02

Data Structures

Organized storage: arrays/lists (ordered) and dictionaries (key-value). Like closets and shelves. Hold IPs, log entries, scan results.

concept 03

Control Structures

Loops and conditionals direct flow. Like traffic signals. Repeat actions and make decisions based on conditions.

pseudocode

Pseudocode is a simplified, high-level outline of a script's logic and steps, without specific-language syntax, like a rough house sketch before construction: you map the structure and flow without exact measurements. It's a blueprint that breaks complex tasks into manageable steps, useful in the design phase (e.g. outlining the sequence to scan network traffic for anomalies). Its simplicity also makes it easy to communicate the process to team members unfamiliar with a given language, and it allows revisions before committing to code, reducing errors. It's not executable, it's a translator between human logic and machine execution.

data structures

Data structures organize and store information in a script, like the closets, shelves, and cabinets in a house. Common types:

  • Arrays/lists: ordered collections where each element is accessed by position, ideal when order matters (e.g. timestamped log files, IP addresses, scan results).
  • Dictionaries: flexible key-value pairs where each unique key maps to a value, ideal for efficient retrieval by key (e.g. associating IP addresses with their statuses).

The right data structure keeps scripts efficient and organized, essential for automated security processes.

control structures

Control structures define the logic and flow, like traffic signals directing cars. Two main types:

  • Loops (for, while): repeat actions, useful for large data sets or recurring tasks (e.g. scanning thousands of login attempts and flagging suspicious ones), saving time and reducing error.
  • Conditional statements (if, else): make decisions based on data (e.g. if disk usage exceeds 90%, clear temp files or send an alert), adding automated decision-making.

Together they let scripts adapt to varying conditions and handle scenarios without manual intervention.

EXAM TIPS

what to know for the exam

  • Pseudocode = a plain-language plan of logic/steps with no syntax; used in the design phase and for communication.
  • Data structures organize data: arrays/lists are ordered (by index); dictionaries are key-value pairs for lookup.
  • Control structures direct flow: loops repeat actions; conditionals make decisions.
  • These three concepts are language-agnostic, they apply to Bash, PowerShell, and Python alike.
decision rules
  • Scenario outlines logic before coding, no syntax → pseudocode
  • Scenario stores an ordered list of log entries → array/list
  • Scenario maps IPs to statuses for lookup → dictionary (key-value)
  • Scenario repeats an action over many entries → loop (control structure)

Security Boundaries

In zero trust, boundaries are defined by access controls and continuous verification, not by physical walls. Every component, data store, and zone is secured on its own terms.

overview

Security boundaries enforce zero trust by monitoring and validating every cross-network interaction instead of trusting a network perimeter. Three concepts define them: system components are the individual elements (servers, workstations, apps) secured separately, data perimeters are the controlled boundaries around sensitive information wherever it resides, and secure zones are tightly controlled, continuously verified network segments. Zone tools include the screened subnet, bastion host, jump box, and air-gapped network.

concept 01

System Components

Individual infrastructure elements (servers, workstations, apps, routers, databases) each secured separately. No component is trusted automatically.

concept 02

Data Perimeters

Boundaries around sensitive data, enforced with encryption, ACLs, and monitoring, protecting it in transit and at rest wherever it lives.

concept 03

Secure Zones

Tightly controlled, continuously verified network segments. Trusted internal, untrusted external, and the screened subnet (DMZ) between them.

system components

System components are the individual elements that make up an organization's IT infrastructure: servers, workstations, applications, routers, and databases. In a zero trust model, no component is trusted automatically, so each is secured separately within strict boundaries. A web server handling public requests should never have direct access to a sensitive database; that access is mediated through role-based access controls, MFA, and network segmentation.

Components are grouped into zones based on their role, such as credit card processing or user authentication, so security policies can be tailored to each. Credit card systems demand encryption and strict access controls; authentication systems demand strong password management. Dividing the network into role-based zones makes it simpler to manage and lets administrators focus on the requirements of their assigned area.

data perimeters

Data perimeters define the boundaries that protect sensitive information, ensuring only authorized users can access it whether it is stored on-premise or in the cloud. They rely on encryption, access control lists, and monitoring, and protect data both in transit and at rest using tools like DLP.

As organizations move to the cloud, the traditional network perimeter is no longer sufficient, so data access requests must be validated continuously. Organizations often create subzones within their networks to store sensitive data, layering on encryption and monitoring for extra protection.

secure zones

Secure zones are tightly controlled areas designed to protect critical systems and data. The three classic zones are the trusted internal zone, the untrusted external zone, and the demilitarized zone (DMZ), also called a screened subnet, which acts as a buffer between internal and external networks. Web servers in the screened subnet may be reachable from the internet while the internal network stays shielded. Firewalls enforce strict rules between zones, and subzones can be carved out with subnetting, firewall rules, and network access control.

secure zone tools

Bastion host
Hardened server designed to withstand external attacks, placed in an exposed area such as the screened subnet. Runs minimal services and uses strict security policies to reduce the attack surface. It controls and manages external access to more secure systems.
Jump box
Hardened gateway used specifically to provide secure administrative access into a secure zone. Admins must connect through the jump box before reaching protected systems, minimizing the exposure of critical hosts. Bastion host controls external access; jump box controls internal administrative access.
Air-gapped network
Network physically separated from all external connections, the highest level of isolation. Used in highly sensitive environments like nuclear facilities. Data must be physically transferred in or out, and strict malware checks are required before any device is connected.

EXAM TIPS

what to know for the exam

  • DMZ = screened subnet. A buffer zone between the trusted internal network and the untrusted internet, where internet-facing servers live.
  • Bastion host vs jump box: bastion host hardens and controls external access; jump box is a gateway for administrators to reach internal secure systems. This distinction is commonly tested.
  • Air-gapped network = total physical isolation. Highest security, but data must be moved manually with malware checks. Used in nuclear/critical facilities.
  • In zero trust, no system component is trusted automatically. A web server should never have direct access to a sensitive database.
  • Data perimeters protect data in transit and at rest using encryption, ACLs, DLP, and monitoring, regardless of where the data resides.
decision rules
  • Scenario involves a buffer zone hosting internet-facing web servers → screened subnet / DMZ
  • Scenario involves a hardened, minimal-service host absorbing external attacks → bastion host
  • Scenario involves admins connecting through one gateway to manage internal systems → jump box
  • Scenario involves a network with no external connections at all → air-gapped network
  • Scenario involves protecting sensitive data wherever it moves or rests → data perimeter

Deperimeterization

Letting go of the network edge as the line of defense. When users, vendors, and devices connect from anywhere, security has to follow the connection, not the location.

overview

Deperimeterization is the shift away from relying on a traditional flat-network perimeter toward verifying and securing every user and device regardless of location. It acknowledges that the network boundary is no longer a geographic line, because users, clients, and vendors need remote access. Three technologies enable it: SDN centralizes software control of network traffic, SD-WAN extends that across wide geographic areas, and SASE fuses networking and security into one cloud-delivered service.

concept 01

SDN

Software-Defined Networking. Software controls network functions via controllers/APIs. Separates control, data, and management planes for flexibility and automation.

concept 02

SD-WAN

Extends SDN across wide geographic areas. Connects branch offices over cheap internet links, routes intelligently by application, enforces uniform policy.

concept 03

SASE

Secure Access Service Edge. Combines networking and security (firewall, SWG, ZTNA) into one cloud service, securing access from any location.

software-defined networking (SDN)

SDN lets software control network functions, replacing traditional hardware-based methods. Controllers and APIs manage the underlying infrastructure and direct traffic as it moves through the network. SDN separates the network into three planes:

  • Control plane: makes decisions like routing and traffic management
  • Data plane: handles the actual movement of traffic between devices
  • Management plane: provides oversight and lets administrators make changes

In traditional networks these functions live inside physical devices; SDN handles them virtually, enabling flexibility and automation. Organizations can set policies and automate deployment of network components in the cloud for faster setup and greater agility. Models include open SDN, hybrid SDN, and SDN overlays. The major risk: the central SDN controller is a single point of failure, if it is lost or compromised, the entire network can go down. SDN also improves security monitoring with more granular control of traffic.

software-defined wide area network (SD-WAN)

SD-WAN builds on SDN but is specifically designed to manage and optimize connections across wide geographic areas, ideal for organizations with branch offices, remote sites, or distributed workers. Traditionally, connecting branch offices over long distances required expensive dedicated lines; SD-WAN uses cheaper internet connections while maintaining security and performance.

It routes traffic intelligently based on application type and the best available path, prioritizing critical applications like video conferencing while sending less urgent traffic over lower-priority routes. SD-WAN enforces security policies consistently across all sites and centralizes control in the cloud, so administrators manage all locations from one interface. This directly supports deperimeterization: the focus shifts from protecting one geographic perimeter to securing all connections wherever they originate.

secure access service edge (SASE)

SASE is a framework that combines networking and security into a single cloud-based service. It integrates multiple tools, firewalls, secure web gateways, and zero trust network access, into one cloud-delivered service, so organizations manage everything from one central point while applying consistent policy across the network.

For a remote employee, SASE secures and continuously monitors the connection to company systems just as if they were in the office. A key strength is protecting data as it moves between environments (on-premise server to cloud application), applying encryption, monitoring, and authorized-only access to each connection. SASE is central to the deperimeterization shift, where security extends to wherever users and devices are rather than being tied to physical locations.

EXAM TIPS

what to know for the exam

  • Deperimeterization = security follows the user/device, not the network edge. The perimeter is no longer a geographic line.
  • SDN separates control plane (decisions), data plane (traffic movement), and management plane (oversight). Software, not hardware, controls the network.
  • SDN's central controller is a single point of failure: compromise or loss can bring down the whole network.
  • SD-WAN = SDN applied across wide areas. Connects branches over cheap internet, routes by application type, enforces uniform policy.
  • SASE = networking + security in one cloud service. Bundles firewall, secure web gateway, and zero trust network access (ZTNA).
  • SDN is enterprise-scope; SD-WAN is geographic-scope; SASE is cloud-delivered to anywhere.
decision rules
  • Scenario involves centrally controlling network traffic through software and APIs → SDN
  • Scenario asks which SDN plane makes routing decisions → control plane
  • Scenario involves connecting branch offices over inexpensive internet with intelligent routing → SD-WAN
  • Scenario involves unifying firewall, secure web gateway, and ZTNA in one cloud service → SASE
  • Scenario involves securing remote users the same as on-site, from a single cloud platform → SASE

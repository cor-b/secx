Access Management

Continuously verifying and controlling access based on policy and context, instead of granting trust just because a request came from inside the network.

overview

Access management continuously verifies and controls user and device access based on policies and contextual factors, which is far more secure than assuming trust by network location. Three concepts define it: subject-object relationships set the rules for how users interact with resources, continuous authorization re-evaluates permissions throughout a session, and context-based reauthentication prompts users to verify again when conditions like location or connection security change.

concept 01

Subject-Object Relationships

Rules defining how subjects (users, devices, apps) may interact with objects (data, files, systems), based on permissions, roles, and access levels.

concept 02

Continuous Authorization

Ongoing evaluation of permissions throughout a session, not just at login. Adapts dynamically when behavior or context shifts.

concept 03

Context-Based Reauthentication

Prompts re-verification when monitored conditions change: new location, switching from secure to insecure connection, or accessing sensitive data.

subject-object relationships

Defining subject-object relationships is the foundation of access management. It sets the rules for how subjects (users, devices, or applications) can interact with objects (data, files, or systems), based on permissions, roles, and access levels in an access control system.

Think of a library: books are the objects and readers are the subjects. Access management specifies who can check out which books. A librarian with higher permissions can access all books, including restricted or rare collections, while a regular reader can access only general, non-restricted titles. The clear definition ensures each subject only interacts with objects it is authorized to access.

continuous authorization

Continuous authorization evaluates access permissions throughout a user's session rather than checking only once at login. Continuous monitoring adapts to changes in context or security posture, enforcing policies dynamically as behavior shifts. In a Windows domain, it is implemented with tools like Group Policy and Active Directory, which continuously evaluate permissions and enforce rules in real time.

For example, a finance team member normally accesses financial reports. If that user suddenly tries to open unrelated or restricted files (HR records, executive-level data), the system detects the deviation, can dynamically restrict access, logs the suspicious activity, and notifies administrators. Access rights continually adapt to the user's behavior.

context-based reauthentication

Context-based reauthentication adds a layer of security by prompting users to verify their identity again when contextual factors change, such as a location shift or a move to an insecure connection. In a Windows environment it is implemented with Conditional Access policies in Azure Active Directory and Microsoft Endpoint Manager.

For example, if an employee working in the office moves to a coffee shop, the system detects the change in location and network. Because accessing resources from a public place is riskier, it prompts the employee to log on again to confirm their identity. This adaptive approach keeps access appropriate in new or risky situations.

EXAM TIPS

what to know for the exam

  • Access management verifies and controls access by policy and context, not by network location. This is the zero trust core.
  • Subject-object relationships define who (subject) can do what to which resource (object). Subjects = users/devices/apps; objects = data/files/systems.
  • Continuous authorization re-evaluates permissions throughout the session, not just at login. Group Policy and Active Directory enforce it in a Windows domain.
  • Context-based reauthentication triggers a fresh login when context changes (new location, insecure connection). Azure AD Conditional Access and Microsoft Endpoint Manager implement it.
  • This lesson reinforces access_auth_authz: conditional access, PDP/PEP, and continuous validation are the same ideas framed for zero trust.
decision rules
  • Scenario involves defining which users can interact with which resources → subject-object relationships
  • Scenario involves access being re-checked continuously during a session as behavior changes → continuous authorization
  • Scenario involves a user being prompted to log in again after moving to public wifi → context-based reauthentication
  • Scenario names Group Policy and Active Directory evaluating access in real time → continuous authorization
  • Scenario names Azure AD Conditional Access triggering on a location change → context-based reauthentication

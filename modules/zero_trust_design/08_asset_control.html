Asset Control

You cannot secure what you cannot see. Maintaining a continuous inventory of every device, app, and data source, then constantly validating that each stays compliant.

overview

Asset control maintains a detailed inventory of all assets and continuously validates their security status against policy. Three concepts build it: asset identification catalogs every device, application, and data source to create a complete inventory, asset management is the ongoing work of patching, configuring, and monitoring those assets, and asset attestation verifies each asset's security posture and compliance, quarantining anything that fails until it is fixed.

concept 01

Asset Identification

Catalogs all devices, apps, and data to build a comprehensive, continuously updated inventory using network scanners and cloud/endpoint tools.

concept 02

Asset Management

Ongoing patching, configuration enforcement, and monitoring to keep identified assets secure, compliant, and correctly configured.

concept 03

Asset Attestation

Verifies each asset's security posture and compliance through checks. Failing assets are quarantined or denied access until they comply.

asset identification

Asset identification is the first step: cataloging all devices, applications, and data within the network to create a comprehensive inventory. Organizations use automated discovery tools so the inventory stays current and accurate, which is essential for security and compliance in a zero trust environment.

  • Network scanning: Nmap or Nessus discover connected devices (laptops, desktops, printers, servers)
  • Software inventory: Microsoft System Center Configuration Manager (SCCM) tracks installed applications
  • Cloud/hybrid: AWS Asset Manager or Azure Resource Manager catalog cloud resources like VMs and databases
  • Endpoint: EDR tools like CrowdStrike or SentinelOne provide a continuous endpoint inventory with security-posture details

asset management

Asset management is the ongoing process of monitoring and maintaining identified assets to keep them secure and compliant: applying the latest updates, ensuring correct configuration, and enforcing security policy. In zero trust, this means continuously watching the health and status of each asset to prevent vulnerabilities.

  • Patch management: Windows Server Update Services (WSUS) deploys and verifies updates
  • Configuration management: Ansible, Puppet, or Chef enforce consistent settings across devices
  • Endpoint management: Microsoft Intune or VMware Workspace ONE monitor compliance and auto-remediate issues
  • Centralized platforms: ServiceNow or SolarWinds track assets and integrate with security tools for real-time status

asset attestation

Asset attestation verifies the security posture and compliance of assets to ensure they meet required standards, through regular or real-time assessments confirming devices, applications, and data are secure, correctly configured, and policy-compliant.

  • EDR: CrowdStrike or SentinelOne continuously monitor for compliance, detecting missing updates or unauthorized configurations and triggering alerts or corrective action
  • SIEM: Splunk or IBM QRadar analyze security data for real-time insight into compliance failures and threats
  • Compliance scanning: Tenable or Qualys run continuous vulnerability scans and compliance checks against benchmarks like CIS

In practice, an organization continuously attests devices, checking for up-to-date antivirus, encryption, and required configurations. If a device fails attestation, it may be quarantined or denied access to sensitive resources until it meets the requirements, ensuring only secure, compliant assets operate in the network.

EXAM TIPS

what to know for the exam

  • Three asset control concepts in order: identification (catalog it), management (maintain it), attestation (verify it).
  • Asset identification = building the inventory with discovery tools (Nmap/Nessus for network, SCCM for software, cloud managers, EDR).
  • Asset management = ongoing patching (WSUS), configuration enforcement (Ansible/Puppet/Chef), and endpoint compliance (Intune).
  • Asset attestation = verifying posture/compliance. A device failing attestation is quarantined or denied access until it complies.
  • EDR (CrowdStrike, SentinelOne) appears in both identification and attestation. Compliance scanners (Tenable, Qualys) check against CIS benchmarks.
  • Continuous inventory and attestation are essential to zero trust: an unknown or non-compliant asset is a risk.
decision rules
  • Scenario involves discovering and cataloging all devices on the network → asset identification (Nmap/Nessus)
  • Scenario involves keeping assets patched and correctly configured → asset management (WSUS, Ansible/Puppet/Chef)
  • Scenario involves verifying a device meets policy before granting access → asset attestation
  • Scenario involves quarantining a device that lacks current antivirus → asset attestation failure
  • Scenario involves scanning assets against CIS benchmarks → compliance scanning (Tenable/Qualys)

Segmentation

Dividing a network into controlled zones to limit access and shrink the attack surface, then taking it further with micro-segmentation to isolate individual workloads.

overview

Segmentation divides a network into distinct zones to control and limit access between areas, reducing the attack surface. It starts with data topology mapping to classify data into zones, then applies boundaries through cloud constructs (VPC/VNet, subnets, security groups, NACLs) and internal environments (production, staging, guest). Micro-segmentation takes it further, isolating individual workloads so a breach in one cannot move laterally to the rest of the network.

level 01

Segmentation

Divides the network into zones (financial, proprietary, guest) to limit access between areas. Reduces attack surface and focuses security on highest-need areas.

level 02

Micro-segmentation

Smaller, granular zones within data centers and cloud, isolating individual workloads. A breached web server stays confined and cannot reach the database.

data topology mapping

Before segmenting networks and servers, organizations map out their data zones through data topology mapping, which defines and classifies data by its users, constraints, flow, and importance. Categorizing data into zones (financial, proprietary) lets the organization apply tailored protections to each. Financial data may need to comply with PCI DSS, while proprietary data may have custom protection requirements. Segmentation then focuses security effort where it is needed most.

segmentation approaches

cloud segmentation

Region-based segmentation
Cloud providers let you store data in specific geographic regions. Useful for multinational organizations that must keep data within regional regulatory boundaries.
Availability zones
Multiple isolated zones within a region provide redundancy and failover. If one zone fails, others stay online to keep critical services available, a form of network segmentation.
VPC / VNet
Virtual Private Cloud (AWS) or Virtual Network (Azure) create private, segmented networks in the cloud. Subnets, security groups, and network ACLs control traffic flow in and out. NACLs apply at the subnet level with allow/deny rules, similar to access control lists.

internal environment segmentation

  • Production: the live systems used by customers
  • Staging: a testing environment used before changes go live
  • Guest: limited-access segment for visitors, without exposing internal resources

peer-to-peer segmentation

Peer-to-peer connections let devices connect directly, but they are less common in enterprises because they lack the granular control of advanced segmentation and are hard to secure. When P2P communication is necessary, it can be segmented using a VLAN in traditional networks, or a VPC/VNet in cloud environments.

micro-segmentation

Micro-segmentation creates smaller zones within data centers and cloud environments to isolate workloads from each other and secure them individually. Administrators set policies that limit traffic between parts of the network based on server roles, application tags, and security templates, following a zero trust approach.

The defining benefit is containment: even if an attacker breaches one system, they are confined to that micro-segment and cannot move freely. If an attacker gains access to a web server, micro-segmentation keeps them isolated from the database and other critical systems, reducing widespread damage and making incidents easier to contain. Micro-segmentation also provides full accountability with an automatic audit trail for every action.

EXAM TIPS

what to know for the exam

  • Segmentation divides a network into zones to limit access and reduce attack surface. Micro-segmentation isolates individual workloads.
  • Micro-segmentation is the zero trust answer to lateral movement: a breached web server cannot reach the database.
  • Data topology mapping comes first: classify data by users, constraints, flow, and importance before segmenting.
  • VPC (AWS) and VNet (Azure) provide cloud network segmentation. NACLs filter at the subnet level with allow/deny rules.
  • Availability zones provide redundancy/failover within a region; region-based segmentation keeps data in a geography for compliance.
  • Production, staging, and guest are internal environment segments. Guest gives visitors limited access without exposing internal resources.
decision rules
  • Scenario involves isolating individual workloads so a breach cannot spread laterally → micro-segmentation
  • Scenario involves separating sensitive financial systems from general user areas → segmentation
  • Scenario involves classifying data before applying protections → data topology mapping
  • Scenario involves a private segmented network in AWS or Azure → VPC / VNet
  • Scenario involves subnet-level allow/deny traffic filtering in the cloud → network ACL
  • Scenario involves keeping data within a specific country for compliance → region-based segmentation

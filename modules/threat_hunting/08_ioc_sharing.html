IoC Sharing

Exchanging indicators of compromise so one organization's discovery protects many. A standard language (STIX), a transport protocol (TAXII), and a government real-time program (AIS).

overview

IoC sharing is the exchange of data about potential threats (malicious IPs, file hashes) between organizations to improve detection and response. Three concepts: STIX (a standardized language for representing threat info), TAXII (the protocol for securely sharing it), and AIS (a US government program for real-time public/private indicator sharing).

concept 01

STIX

Structured Threat Information eXpression: a standardized LANGUAGE/format for threat data (IOCs, vulnerabilities, TTPs).

concept 02

TAXII

Trusted Automated eXchange of Intelligence Information: the PROTOCOL for securely, automatically sharing threat data.

concept 03

AIS

Automated Indicator Sharing: a US DHS PROGRAM for real-time public/private IOC sharing at machine speed.

STIX, TAXII, and AIS

STIX (Structured Threat Information eXpression)
A standardized language to represent and communicate cyber threat information (IOCs, vulnerabilities, attacker tactics) in a consistent, structured format, so data can be shared and understood across organizations and tools. Integrates with SIEMs and TIPs (e.g. banks formatting phishing data in STIX to share).
TAXII (Trusted Automated eXchange of Intelligence Information)
The protocol that securely and automatically exchanges threat intelligence over the internet, enabling real-time updates without manual effort. TAXII-compliant tools pull in new IOCs (e.g. hospitals sharing ransomware-strain info).
AIS (Automated Indicator Sharing)
A US Department of Homeland Security program enabling real-time IOC exchange (malicious IPs, hashes, domains) between public and private sectors at machine speed, giving access to up-to-date threat info from government and other companies.

Memory hook: STIX = the language/format, TAXII = the transport/protocol, AIS = the government program. They combine: format data in STIX, share it via TAXII, and participate in AIS for real-time alerts. (Note: the lesson demoed pulling AlienVault OTX data via REST API into JSON when TAXII isn't supported, then noted a script could convert it to STIX if needed.)

EXAM TIPS

what to know for the exam

  • STIX = the standardized language/format for threat data; TAXII = the protocol that transports it.
  • Don't swap them: STIX describes, TAXII delivers.
  • AIS = a US DHS program for real-time public/private IOC sharing at machine speed.
  • IoCs include malicious IPs, file hashes, and domain names.
  • Workflow: format in STIX → share via TAXII → AIS enables real-time gov/industry exchange.
decision rules
  • Scenario formats threat data in a standardized language → STIX
  • Scenario securely transports threat data between orgs → TAXII
  • Scenario shares IOCs in real time with the government → AIS
  • Scenario exchanges malicious IPs/hashes to improve detection → IoC sharing

Detection and Threat-hunting Enablers

The infrastructure that makes hunting possible: where you watch (sensors), watching all the time (continuous monitoring), getting notified (alerting), and pooling it all in one place (centralized logging).

overview

Detection and threat-hunting enablers are the tools and infrastructure that facilitate finding and investigating threats. Four concepts: sensor placement (positioning monitoring devices), continuous monitoring (constant real-time tracking), alerting (notifications on predefined criteria), and centralized logging (aggregating logs into one platform).

enabler 01

Sensor Placement

Strategically position IDS/IPS sensors (perimeter, internal segments) to capture critical traffic, including east-west. Zeek (formerly Bro).

enabler 02

Continuous Monitoring

Constant real-time watch over network/systems/endpoints; spots deviations instantly. SIEM (Splunk).

enabler 03

Alerting

Predefined rules/thresholds trigger notifications (failed logins, odd traffic). Like a fire alarm. Splunk, Palo Alto.

enabler 04

Centralized Logging

Aggregate logs from all sources into one platform (ELK Stack) for correlation, a single hub.

sensor placement and continuous monitoring

Sensor placement is the strategic deployment of monitoring devices (IDS/IPS) across the network, perimeter (between corporate network and internet) and internal segments (server zones capturing east-west traffic, communication between internal devices), like surveillance cameras at building entry points. Tool: Zeek (formerly Bro). Without proper placement, attackers slip through undetected. Continuous monitoring maintains a constant, real-time watch over the network, systems, and endpoints (rather than periodic reviews), flagging unauthorized access or deviations instantly, like a guard watching surveillance feeds (tool: Splunk/SIEM).

alerting and centralized logging

Alerting sets predefined rules or thresholds that trigger notifications on suspicious events (multiple failed logins, unusual outbound traffic, odd-location access), like a fire alarm, it doesn't stop the threat but gives the team a heads-up to react (tools: Splunk, Palo Alto Networks). Centralized logging aggregates log data from all devices, servers, applications, and security tools into one location, giving a complete view and enabling correlation across the network. It's like a central hub for all the building's camera footage (tool: ELK Stack, Elasticsearch + Logstash + Kibana).

EXAM TIPS

what to know for the exam

  • Four enablers: sensor placement, continuous monitoring, alerting, centralized logging.
  • Sensor placement captures critical traffic including east-west (internal device-to-device); tool: Zeek.
  • Continuous monitoring = constant real-time watch (not periodic); alerting = predefined-rule notifications (fire alarm).
  • Centralized logging aggregates all logs into one platform (ELK Stack) for correlation.
  • East-west traffic = lateral communication inside the network; north-south = in/out of the network.
decision rules
  • Scenario positions IDS sensors to capture internal device-to-device traffic → sensor placement (east-west)
  • Scenario watches the network constantly in real time → continuous monitoring
  • Scenario notifies the team on multiple failed logins → alerting
  • Scenario funnels all logs into one platform to correlate → centralized logging

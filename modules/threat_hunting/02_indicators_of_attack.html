Indicators of Attack

Behaviors that signal an attack in progress, organized by the attacker's playbook (TTPs). Catch them at each stage, from getting in to staying in.

overview

Indicators of attack (IoA) are observable behaviors and patterns suggesting an ongoing or imminent attack, focused on the attacker's tactics, techniques, and procedures (TTPs). Tactics = the goal, techniques = the actions, procedures = the exact methods. Recognizing TTPs lets hunters detect malicious activity and predict the next step. The lesson follows six TTP stages.

following the TTPs

  1. initial access First penetration: phishing, exploiting vulnerabilities, stolen credentials. Detect via login-attempt surges from unfamiliar IPs, phishing reports. Defend with email filtering, MFA, SIEM.
  2. privilege escalation Elevating access: unpatched software, malicious scripts, modified privileges. Detect via suspicious PowerShell or unusual privilege changes (EDR like CrowdStrike).
  3. lateral movement Moving system to system: RDP, admin tools like PsExec. Detect via unusual RDP connections or admin tools on systems that don't normally communicate (SIEM, network analyzers).
  4. exfiltration Stealing data out: encrypted channels, disguised traffic, legitimate cloud services. Detect via unusual outbound traffic/large transfers (DLP).
  5. command and control (C2) Maintaining communication: DNS tunneling, malicious servers. Detect via unusual outbound connections to suspicious domains/DNS queries (IDS, network monitoring).
  6. persistence Keeping access: new admin accounts, backdoors, modified settings. Detect via unauthorized account creation or config changes (EDR like Carbon Black, CrowdStrike).

Setting up alerts for known TTPs lets teams detect suspicious activity early and respond before escalation, combining EDR and SIEM with threat intelligence to stay ahead.

EXAM TIPS

what to know for the exam

  • IoA = behaviors signaling an attack, based on TTPs (tactics = goal, techniques = actions, procedures = methods).
  • Six TTP stages: initial access, privilege escalation, lateral movement, exfiltration, C2, persistence.
  • Detect privilege escalation via suspicious PowerShell; lateral movement via unusual RDP/PsExec; C2 via odd DNS/outbound connections.
  • Tools: EDR (CrowdStrike, Carbon Black), SIEM, DLP (exfiltration), IDS (C2).
  • IoA focuses on behavior, distinct from IoC, which focuses on artifacts (IPs, hashes).
decision rules
  • Scenario sees a phishing surge gaining a foothold → initial access
  • Scenario detects suspicious PowerShell raising privileges → privilege escalation
  • Scenario sees unusual RDP/PsExec between systems → lateral movement
  • Scenario detects DNS tunneling to a suspicious domain → command and control

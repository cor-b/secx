Rule-Based Languages

Four tools for writing detection rules, each with its own domain: SIEM events (Sigma), files and malware (YARA), network anomalies (Rita), and live network traffic (Snort).

overview

Rule-based languages create patterns and detection rules to identify malicious activity. Four examples: Sigma (generic SIEM event patterns), YARA (file/malware classification), Rita (network anomaly detection), and Snort (real-time network IDS/IPS). Each targets a different layer; they work together.

tool 01

Sigma

Generic, platform-agnostic SIEM event patterns (YAML). Write once, use across SIEMs. E.g. detect a "whoami" command.

tool 02

YARA

Identify/classify malware by patterns in files (strings, sizes, byte sequences). For malware researchers and file scanning.

tool 03

Rita

Open-source network anomaly detection (beaconing, DNS tunneling, long connections). Integrates with Zeek logs.

tool 04

Snort

Open-source network IDS/IPS analyzing real-time traffic with rules/signatures (port scans, exploit attempts).

the four languages

Sigma
A generic, platform-agnostic rule language (written in YAML) for defining security event patterns across SIEMs, write a rule once and use it on any Sigma-supporting SIEM. Example: detect command-line execution of whoami (a common reconnaissance command).
YARA
"Yet Another Recursive Acronym": identifies and classifies malware by defining patterns in files (strings, file sizes, byte sequences). Used in threat hunting and incident response to scan endpoints/files (e.g. matching suspicious strings like whoami, hostname, netstat).
Rita
An open-source framework for detecting network anomalies like beaconing (malware contacting C2 at intervals), DNS tunneling, and long-running connections. Integrates with Zeek (formerly Bro) logs; good for stealthy APTs.
Snort
An open-source network IDS/IPS that analyzes real-time traffic against predefined or custom rules/signatures (port scans, malware payloads, exploit attempts like Shellshock), blocking or alerting. Works alongside Sigma/YARA/Rita.

Layered: Snort detects at the network layer in real time, while Sigma and YARA detect malicious behavior/IOCs at the system/file level, and Rita finds covert network patterns.

EXAM TIPS

what to know for the exam

  • Sigma = generic, platform-agnostic SIEM event rules (YAML); write once, use across SIEMs.
  • YARA = identify/classify malware by file patterns (strings, byte sequences).
  • Rita = network anomaly detection (beaconing, DNS tunneling); integrates with Zeek.
  • Snort = real-time network IDS/IPS with rules/signatures.
  • Match the domain: Sigma → SIEM events, YARA → files/malware, Rita → network anomalies, Snort → live traffic.
decision rules
  • Scenario writes one rule usable across any SIEM → Sigma
  • Scenario classifies malware by patterns in files → YARA
  • Scenario detects C2 beaconing in network flow → Rita
  • Scenario blocks an exploit attempt in real-time traffic → Snort

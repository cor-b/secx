Data Labeling and Tagging

Labels communicate classification level. Tags add finer-grained handling requirements. Together they ensure data is protected according to its sensitivity, regardless of who encounters it.

overview

Classification determines a data sensitivity tier; labeling and tagging make that classification visible and actionable. Labels are broad classification markers (confidential, secret, top secret). Tags add specific handling requirements within or alongside a classification (PII, PHI, BIGOT). Tags exist at every classification level, including unclassified data that still requires protection.

data labels

A data label marks a piece of information with its classification level so that anyone who encounters it knows how to handle and protect it. In military documents, the classification label appears in both the header and footer of every page.

manual vs automatic labeling

Manual labeling
The user creating the data applies the classification label themselves. For example, an employee writing a document containing the company's new marketing strategy decides it should only be seen by top-level executives and labels it as confidential or top secret. Manual labeling relies on the creator's judgment and knowledge of classification policy.
Automatic labeling
The system applies classification labels based on predefined keyword rules, sometimes called a dirty word list. If a user types a word that appears on the dirty word list, the document is automatically labeled at the corresponding classification level. For instance, if "bazooka" is a secret-classified term, any document containing it is automatically labeled secret — regardless of whether the author meant the weapon or the chewing gum brand. This ambiguity is the limitation of automatic labeling: the system cannot understand context.

declassification

Labels can also carry declassification requirements, indicating when the data may be downgraded to a lower classification or made public. Declassification is triggered when the data is no longer sensitive enough to warrant its current level of protection.

Example: the plans for the Allied invasion of Normandy in June 1944 (Operation Overlord) were labeled top secret during World War II. After the invasion, the information was no longer sensitive, so it was declassified and is now publicly available. Declassifying old data frees up the resources used to protect it, which can then be redirected to protecting current sensitive data.

data tags

Tags provide a level of specificity that classification labels cannot. While a label indicates how sensitive data is, a tag describes what the data contains and what special handling rules apply to it. Tags exist at every classification level, including unclassified.

common industry tags

PII
Personally Identifiable Information — data that can identify a specific individual. Examples include names, addresses, Social Security numbers, and email addresses. PII can appear on otherwise unclassified documents but still requires protection to prevent identity theft or privacy violations.
SPI
Sensitive Personal Information — a subset of PII considered more sensitive. SPI typically includes information such as financial data, biometric data, or data about protected characteristics. It carries stricter handling requirements than general PII.
PHI
Protected Health Information (or Personal Health Information) — medical and health-related data. A hospital patient's medical record is not classified as top secret, but the PHI tag indicates it must still be protected. HIPAA legally mandates specific handling requirements for PHI, making the tag a compliance driver rather than just an internal policy.
Financial / Restricted
Tags indicating data subject to financial regulations or restricted to specific roles or departments. Used to flag data governed by regulations such as SOX (Sarbanes-Oxley) or PCI DSS. The tag ensures automated controls like DLP can enforce access restrictions even when the document itself carries a lower classification label.

tags within classified data

Tags also exist within highly classified tiers to add further specificity beyond the label alone. The Operation Overlord documents, for example, were labeled top secret but also carried the tag BIGOT (British Invasion of German-Occupied Territory), requiring additional special handling beyond what the top secret label alone mandated.

Within the top secret classification, examples of further sub-tags include SI (Special Intelligence), TK (Talent Keyhole), and HCS (Human Intelligence Control System). These are not exam-required but demonstrate that the tag system applies at every level of classification.

structured vs unstructured data

Labels and tags must be applied to data regardless of its format. Because different formats require different technical approaches to classification and detection, understanding the distinction between structured and unstructured data matters for DLP configuration.

TypeDefinitionExamples
Structured Organized according to a predefined model; each field is predictable and machine-parseable CSV (comma-separated values), JSON, SQL database records
Unstructured No fixed format; content is free-form and variable in length and layout Word documents, emails, PowerPoint files, text files, chat logs

DLP systems and classification engines must handle both formats. Structured data can be parsed predictably by field; unstructured data requires content inspection and pattern matching to identify sensitive information.

exam tips

key takeaways

  • Labels indicate classification level; tags indicate specific handling requirements. Labels are broad (top secret). Tags are specific (PHI, PII, BIGOT). A document can have one label and multiple tags. Both apply simultaneously.
  • Tags exist at every classification level, including unclassified. A medical record is unclassified but carries the PHI tag. The tag is what triggers legal protection requirements, not the classification label.
  • Automatic labeling uses dirty word lists; manual labeling is user-applied. Automatic labeling is efficient but context-blind (it cannot distinguish "bazooka the weapon" from "Bazooka the brand"). Manual labeling is accurate but depends on the creator's judgment.
  • Declassification is when data is downgraded because it is no longer sensitive. The Operation Overlord example illustrates this: top secret during the war, unclassified after. Declassification frees up protection resources for more current data.
  • Military documents carry the classification label in both the header and footer. This ensures the classification is visible on every page, not just the cover.
  • Structured data (CSV, JSON) vs unstructured data (Word, email) require different DLP handling. Structured data is parsed field by field; unstructured data requires content inspection.
quick decision rules
  • Classification marker indicating sensitivity tier (confidential, top secret) → Data label
  • Specific handling requirement within or alongside a label (PII, PHI, BIGOT) → Data tag
  • System automatically applies label because a keyword triggered it → Automatic labeling (dirty word list)
  • User applies the label themselves at document creation → Manual labeling
  • Top secret data downgraded after it is no longer sensitive → Declassification
  • Unclassified medical record that still requires legal protection → PHI tag (HIPAA)
  • Comma-separated or JSON file → Structured data
  • Word document, email, or PowerPoint → Unstructured data

Cloud Security

Securing the cloud end to end: how you adopt it, who's responsible for what, how you connect and integrate, and how you build, deploy, and run workloads safely.

overview

Domain 2, Objective 2.5: securely implement cloud capabilities in an enterprise environment. Securing the cloud spans implementation, management, connectivity, and the development/deployment pipeline, with security embedded at every stage, from shared responsibility and API protection to containers and serverless.

area 01

Adopt & Manage

Cloud implementation (control strategies), cloud management (shared responsibility, keys/licenses), connectivity and integration.

area 02

Protect

Cloud security considerations (storage, exposure, leakage, remnants), API security, and Cloud Access Security Brokers (CASB).

area 03

Build & Run

Development and deployment (Terraform, Ansible), CI/CD pipeline, container management, and serverless computing.

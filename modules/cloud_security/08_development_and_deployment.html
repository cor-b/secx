Development and Deployment

Building and shipping cloud infrastructure as code: defining it with Terraform, configuring it with Ansible, and watching the third-party packages you pull in.

overview

Secure cloud development and deployment rests on infrastructure as code (IaC), defining infrastructure in version-controlled config files for consistent, repeatable provisioning, plus package monitoring to track third-party dependencies for vulnerabilities. Key tools: Terraform (provisioning) and Ansible (configuration management).

tool 01

Terraform

Infrastructure as code: declaratively provision cloud resources from version-controlled config. Consistent, repeatable, auditable.

tool 02

Ansible

Configuration management: automate setup, config, and patching of provisioned systems to a known good state.

tool 03

Package Monitoring

Track third-party libraries/dependencies for known vulnerabilities and keep them patched (supply-chain risk).

infrastructure as code (IaC)

IaC defines and provisions infrastructure through machine-readable config files instead of manual setup, so environments are consistent, repeatable, and version-controlled. Because the config lives in source control, changes are reviewed, auditable, and easy to roll back, and the same definition reliably reproduces dev, test, and prod. Terraform is the standard tool: you declare the desired end state (networks, compute, storage) and it provisions resources across providers to match.

configuration management with ansible

Once infrastructure is provisioned, Ansible automates configuring it, installing software, applying settings, and patching, so systems converge to a defined, secure baseline. Automating configuration removes manual error and drift, and applies security hardening uniformly across many machines. Used alongside Terraform: Terraform builds the resources, Ansible configures what runs on them.

package monitoring

Modern apps depend heavily on third-party libraries and packages, each a potential entry point if it contains a known vulnerability (a supply-chain risk). Package monitoring continuously tracks these dependencies, flags packages with disclosed CVEs, and prompts updates to patched versions. Keeping dependencies current and scanned is essential to securing the deployment pipeline.

EXAM TIPS

what to know for the exam

  • IaC = infrastructure defined in version-controlled config files: consistent, repeatable, auditable provisioning.
  • Terraform provisions infrastructure (declarative); Ansible configures and patches what's provisioned.
  • Remember the split: Terraform builds resources, Ansible configures them.
  • Package monitoring tracks third-party dependencies for vulnerabilities, a supply-chain defense.
decision rules
  • Scenario provisions cloud resources from declarative version-controlled files → Terraform (IaC)
  • Scenario automates configuration and patching of existing servers → Ansible
  • Scenario flags a vulnerable third-party library in the build → package monitoring
  • Scenario wants identical, repeatable environments across dev/test/prod → infrastructure as code
